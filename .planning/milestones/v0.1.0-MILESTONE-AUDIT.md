## Tech Debt

### Phase 7: Gate Evaluation

1. **No CLI integration tests (Moderate)** — `assay-cli` has 0 automated tests. All CLI behaviors (streaming output, exit codes, --json, --verbose, --timeout, summary format) verified only by manual UAT during plan execution.
2. **duration_ms > 0 not asserted (Minor)** — `evaluate_echo_hello` test comment acknowledges "may be 0 on very fast machines" but the success criterion requires non-zero.
3. **Skipped criteria not named in streaming output (Minor)** — Summary shows count ("K skipped") but individual skipped criterion names are not displayed.
4. **Summary format mismatch (Informational)** — Truth says "N/M criteria passed"; actual output is "Results: N passed, M failed, K skipped (of T total)" — richer than stated, intentional.

### Phases 8-10: Missing Formal Verifications

Phases 8, 9, and 10 lack formal VERIFICATION.md reports. All three have completion summaries documenting `just ready` passing and requirements satisfied. The integration checker verified cross-phase wiring independently. This is a process gap, not a functional gap.

### Documentation: Stale REQUIREMENTS.md

REQUIREMENTS.md has 23 requirements (GATE-01 through GATE-08, MCP-01 through MCP-08, PLG-01 through PLG-07) still marked `[ ]` despite being functionally complete. The traceability table also shows "Not Started" for these. Should be updated to `[x]` / "Complete".
