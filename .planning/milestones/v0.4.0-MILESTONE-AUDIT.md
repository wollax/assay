**Status: COMPLETE** — all 12 steps wired, no broken connections.

### Flow 2: Session lifecycle
`session_create` (Phase 41) → `session_update(AgentRunning)` → `gate_evaluate(session_id=...)` auto-transitions to `GateEvaluated` via `record_gate_result` (Phase 42) → `session_get`/`session_list` shows updated state.

**Status: COMPLETE** — state machine enforced, transitions validated.

### Flow 3: Recovery
`serve()` calls `recover_stale_sessions()` synchronously before accepting connections → threshold from `SessionsConfig` (Phase 42) → stale `AgentRunning` sessions marked `Abandoned`.

**Status: COMPLETE** — recovery runs before tool acceptance.

### Flow 4: spec_validate
MCP tool (Phase 37) → `load_spec_entry_with_diagnostics` → `validate_spec_with_dependencies` (cycles, prompts, commands) → returns `ValidationResult` with diagnostics.

**Status: COMPLETE** — errors surfaced as `ValidationResult`, not tool errors.

### Flow 5: Observability
`warnings` on all mutating responses (Phase 35) → `gate_history` outcome+limit (Phase 35) → `spec_get` resolve (Phase 38) → `estimate_tokens` growth_rate (Phase 38).

**Status: COMPLETE** — all observability surfaces wired.
