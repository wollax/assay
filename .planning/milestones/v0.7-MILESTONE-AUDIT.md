## Tech Debt

### 1. Milestone Pipeline Bypass (pre-existing)

`cycle_advance()` in `crates/assay-core/src/milestone/cycle.rs` and `pr_check_milestone_gates()` in `crates/assay-core/src/pr.rs` call `evaluate_all_gates()` directly without calling `compose::resolve()` or `check_preconditions()`. This means `extends`, `include`, and `[preconditions]` are silently ignored during:
- `assay milestone advance` (CLI)
- `assay pr create` (CLI)
- MCP `cycle_advance` tool
- MCP PR creation path
- TUI `/gate-check` slash command

**Impact:** Low — this code pre-dates v0.7.0 (M005/M018 milestones). Phase 70 explicitly scoped to `handle_gate_run` and MCP `gate_run`. Track for future milestone.

### 2. Pre-existing TODO

`TODO(M024/S02): stream_criterion should accept events slice` in `crates/assay-cli/src/commands/gate.rs:291` — scoped to a future milestone, not introduced by v0.7.0.

### 3. SUMMARY Frontmatter Gap

None of the 19 plan SUMMARY.md files include `requirements_completed` in YAML frontmatter. This is a process consistency gap — requirements coverage is fully tracked in VERIFICATION.md and REQUIREMENTS.md.
