# Milestone v0.3.0: Orchestration Foundation

**Status:** ✅ SHIPPED 2026-03-10
**Goal:** Build the foundation for agent orchestration — worktree isolation, independent gate evaluation infrastructure, and CLI/MCP/types/core hardening — while closing tech debt from v0.2.0.
**Phases:** 26-34
**Total Plans:** 22

## Overview

v0.3.0 established the worktree isolation foundation needed for future agent orchestration while systematically closing tech debt accumulated during v0.2.0's rapid feature development. The milestone covered seven work areas: structural prerequisites, types hygiene, worktree management, gate output truncation, core tech debt, error messages, CLI polish, MCP validation, and MCP truncation visibility.

## Milestone Summary

**Key Decisions:**
- Head/tail truncation ratio 1:2 (33% head, 67% tail) with marker as overhead
- Levenshtein fuzzy match threshold: distance <= 2 AND distance <= name.len() / 2
- SpecNotFoundDiagnostic as separate error variant (not enriching existing SpecNotFound)
- ERR-01 detects exit codes 127/126 since commands spawn via sh -c
- MCP param validation tested at serde layer (rmcp Parameters<T> handles before handler)
- Stderr-first, stdout-fallback chain for MCP failure reason extraction
- CriterionSummary truncation fields use Option types for JSON omission

**Issues Resolved:**
- CLI module extraction unblocked all CLI-facing work (Phase 32)
- Gate output truncation gap between core and MCP surfaces closed (Phase 34)
- ~160 lines of duplicated validation/evaluation code eliminated (Phase 30)

**Issues Deferred:**
- ~35 raw AssayError::Io construction sites not migrated to helpers (intentional)
- plugin.json version 0.1.0 does not match workspace version (pre-existing)
- 42 open issues in .planning/issues/open/ (test gaps, derives, naming, refactors)

**Technical Debt Incurred:**
- Raw AssayError::Io construction sites (existing code stays, new code uses helpers)
- Plugin version mismatch (pre-existing from v0.1.0)
