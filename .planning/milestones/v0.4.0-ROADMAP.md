# Milestone v0.4.0: Headless Orchestration

**Status:** ✅ SHIPPED 2026-03-15
**Goal:** Ship `gate_evaluate` as the capstone MCP tool — agent-evaluated gates driven by headless subprocess orchestration, backed by session persistence, context-aware diff budgeting, spec validation, and observability improvements.
**Phases:** 35-45
**Total Plans:** 30

## Overview

v0.4.0 delivers the headless orchestration layer that makes Assay's dual-track gates fully autonomous. The capstone `gate_evaluate` MCP tool computes diffs, spawns headless Claude Code evaluators as subprocesses, parses structured per-criterion results, and persists GateRunRecords — all in a single call. This is backed by `WorkSession` persistence for tracking agent work across phases, external `cupel` context engine integration for token-budgeted context slicing, `spec_validate` for static spec health checking, and comprehensive observability/correctness improvements.

## Milestone Summary

**Key Decisions:**
- `gate_evaluate` uses subprocess model — parent parses structured JSON output, evaluator never calls MCP tools
- `EvaluatorOutput` schema defined before prompt engineering — lenient `serde_json::Value` intermediate parse
- `WorkSession` (on-disk) is distinct from `AgentSession` (in-memory v0.3.0)
- Session management within `gate_evaluate` is Rust function calls, not MCP round-trips
- Context engine is external crate (`cupel`, separate repo), not workspace crate
- `spec_validate` check_commands is opt-in (off by default)
- Evaluator schema cached with `LazyLock` (generated once per process)
- `EvaluateCriterionResult` uses typed `CriterionOutcome`/`Enforcement` enums
- `DiffTruncation` byte fields use `u64` for platform-independent serialization
- `SessionPhase` is `#[non_exhaustive]` for forward compatibility
- Recovery scan capped at 100 sessions (oldest first via ULID sort)

**Issues Resolved:**
- 120+ tech debt issues closed across types, core, CLI, and MCP crates
- History save failure now surfaced via warnings field (OBS-01)
- False 0/0 worktree ahead/behind fixed for assay-managed branches

**Issues Deferred:**
- 122 open issues in .planning/issues/open/ (non-blocking tech debt carried forward)
- cargo-deny warns about 2 unmatched license allowances (cosmetic)

**Technical Debt Incurred:**
- Guard daemon issues deferred to dedicated cleanup phase (18 issues)
