    **Step 1: Add source_tag helper function**

    Add a helper near the top of the file (after the existing helper functions):
    ```rust
    /// Format a source annotation tag for CLI streaming output.
    /// Returns empty string for Own criteria (per CONTEXT.md: signal-to-noise tradeoff).
    fn source_tag(source: Option<&assay_types::resolved_gate::CriterionSource>) -> String {
        use assay_types::resolved_gate::CriterionSource;
        match source {
            Some(CriterionSource::Parent { gate_slug }) => format!(" [Parent: {gate_slug}]"),
            Some(CriterionSource::Library { slug }) => format!(" [Library: {slug}]"),
            Some(CriterionSource::Own) | None => String::new(),
        }
    }
    ```

    **Step 2: Update stream_criterion signature**

    Add `source: Option<&assay_types::resolved_gate::CriterionSource>` parameter to `stream_criterion()`. Update the eprint line from:
    ```rust
    eprint!("{cr}  {label} {} ... running", criterion.name);
    ```
    to:
    ```rust
    let tag = source_tag(source);
    eprint!("{cr}  {label} {}{tag} ... running", criterion.name);

Update all existing callers of stream_criterion to pass `None` for the source parameter (existing Legacy path and handle_gate_run_all Legacy path).

    **Step 3: Add save_precondition_blocked_record helper**

    Add a helper function to save a precondition-blocked record to history:
    ```rust
    fn save_precondition_blocked_record(
        assay_dir: &Path,
        spec_name: &str,
        working_dir: &Path,
        status: &assay_types::PreconditionStatus,
        max_history: Option<usize>,
    ) {
        let summary = assay_types::GateRunSummary {
            spec_name: spec_name.to_string(),
            results: Vec::new(),
            passed: 0,
            failed: 0,
            skipped: 0,
            total_duration_ms: 0,
            enforcement: assay_types::EnforcementSummary::default(),
        };
        let timestamp = chrono::Utc::now();
        let run_id = assay_core::history::generate_run_id(&timestamp);
        // Note: generate_run_id may be pub(crate). If so, construct manually or
        // use the existing pattern. Alternative: build the record and call save() directly.
        let record = assay_types::GateRunRecord {
            run_id: /* generate ID */,
            assay_version: env!("CARGO_PKG_VERSION").to_string(),
            timestamp,
            working_dir: Some(working_dir.display().to_string()),
            summary,
            diff_truncation: None,
            precondition_blocked: Some(true),
        };
        match assay_core::history::save(assay_dir, &record, max_history) {
            Ok(result) => {
                if result.pruned > 0 {
                    tracing::info!(pruned = result.pruned, spec_name = %spec_name, "Pruned old run(s)");
                }
            }
            Err(e) => {
                tracing::warn!(error = %e, "Could not save precondition-blocked run history");
            }
        }
    }
    ```

    **Step 4: Modify handle_gate_run() for SpecEntry::Directory**

    In `handle_gate_run()`, replace the `SpecEntry::Directory` branch in both JSON and streaming paths:

    **JSON mode:**
    ```rust
    SpecEntry::Directory { gates, slug, .. } => {
        // Step 1: Resolve extends + include
        let specs_dir_clone = specs_dir.clone();
        let assay_dir_clone = assay_dir.clone();
        let resolved = assay_core::spec::compose::resolve(
            &gates,
            &slug,
            |parent_slug| {
                let path = specs_dir_clone.join(parent_slug).join("gates.toml");
                assay_core::spec::load_gates(&path)
            },
            |lib_slug| {
                assay_core::spec::compose::load_library_by_slug(&assay_dir_clone, lib_slug)
            },
        )?;

        // Step 2: Check preconditions
        if let Some(preconditions) = &gates.preconditions {
            let assay_dir_for_prec = assay_dir.clone();
            let status = assay_core::gate::check_preconditions(
                preconditions,
                |s| assay_core::history::last_gate_passed(&assay_dir_for_prec, s),
                &working_dir,
                cli_timeout,
                config_timeout,
            );
            if !status.all_passed() {
                // Print blocked status as JSON
                let outcome = assay_types::GateEvalOutcome::PreconditionFailed(status);
                let output = serde_json::to_string_pretty(&outcome)
                    .context("failed to serialize precondition status")?;
                println!("{output}");
                save_precondition_blocked_record(&assay_dir, name, &working_dir, max_history);
                return Ok(2);
            }
        }

        // Step 3: Evaluate resolved criteria
        assay_core::gate::evaluate_all_resolved(
            &slug,
            &resolved.criteria,
            gates.gate.as_ref(),
            &working_dir,
            cli_timeout,
            config_timeout,
        )
    }
    ```

    **Streaming mode:**
    For the streaming path, replace `spec_entry_gate_info()` usage for Directory entries. After resolution and precondition check, iterate over `resolved.criteria` calling `stream_criterion()` with each `ResolvedCriterion.criterion` and `Some(&rc.source)`.

    The streaming path for SpecEntry::Directory becomes:
    1. Resolve via `compose::resolve()`
    2. Check preconditions — if blocked, print "Preconditions not met" message, save blocked record, return exit code 2
    3. Loop over `resolved.criteria`, call `stream_criterion(&rc.criterion, ..., Some(&rc.source))`
    4. Save streaming summary as before

    Legacy specs continue to use `spec_entry_gate_info()` and `stream_criterion(..., None)`.

    **Step 5: Modify handle_gate_run_all()**

    Apply the same resolve + precondition pipeline for each Directory entry. In the `--all` path:
    - For each `SpecEntry::Directory`, resolve and check preconditions
    - If precondition blocked: save blocked record, print a "blocked" message for that spec, continue to next spec (do NOT return exit code 2 — continue processing remaining specs, per CONTEXT.md "existing behavior continues on individual failures")
    - Track a separate `blocked_count` counter for the final summary
    - JSON mode: collect GateEvalOutcome (either Evaluated or PreconditionFailed) per spec
    - Final exit code: 2 if any spec was precondition-blocked and no spec failed (gate failure takes priority), 1 if any required criterion failed, 0 if all passed

    **Step 6: Update gate_exit_code or replace with new exit code logic**

    For the single-spec path, the exit code is determined before calling `gate_exit_code`. For the `--all` path, update the logic:
    - If any counters.gate_blocked() → 1
    - Else if blocked_count > 0 → 2
    - Else → 0
