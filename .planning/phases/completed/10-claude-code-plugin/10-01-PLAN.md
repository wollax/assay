Create `plugins/claude-code/skills/gate-check/SKILL.md` with YAML frontmatter:
- `name: gate-check`
- `description`: "Run quality gates for a spec and report pass/fail results. Use when checking if code changes meet spec criteria, after implementing features, or when asked about gate status."
- Body instructions:
  1. If spec name provided as `$ARGUMENTS`, use it. Otherwise call `spec_list` MCP tool to discover specs.
  2. Call `gate_run` MCP tool with spec name, `include_evidence: false` for initial check.
  3. Report results: all passed = concise summary ("3/3 criteria passed for [spec-name]"); any failed = list each failed criterion with reason, then call `gate_run` with `include_evidence: true` for failures.
  4. If multiple specs and none specified, run gates for ALL specs with aggregate summary.
- Output format guidance: concise for passing, detailed for failures.

Follow the exact SKILL.md format from 10-RESEARCH.md (Pattern 1: Skills as MCP Tool Orchestrators).

Create `plugins/claude-code/skills/spec-show/SKILL.md` with YAML frontmatter:
- `name: spec-show`
- `description`: "Display a spec's criteria and details. Use when the user wants to see what a spec contains, what criteria need to be met, or before starting implementation."
- Body instructions:
  1. If spec name provided as `$ARGUMENTS`, use it. Otherwise call `spec_list` to show available specs and ask which to display.
  2. Call `spec_get` MCP tool with spec name.
  3. Present: spec name, description, each criterion with name/description/executable vs descriptive/command/timeout.
- Output format: structured, group by type (executable vs descriptive).

Follow the exact SKILL.md format from 10-RESEARCH.md.

Remove the `.gitkeep` file from `plugins/claude-code/skills/` since real files now exist.
</action>
