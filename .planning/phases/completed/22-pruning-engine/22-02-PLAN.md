<implementation>
Create `crates/assay-core/src/context/pruning/strategies/progress_collapse.rs`:

```rust
pub fn progress_collapse(
    entries: Vec<ParsedEntry>,
    _tier: PrescriptionTier,
    protected: &HashSet<usize>,
) -> StrategyResult
```

Logic:
- Iterate entries
- If entry is `SessionEntry::Progress(_)` AND line_number not in protected set:
  - Add to bytes_saved, increment lines_removed
  - If samples.len() < 3, push a PruneSample
  - Do NOT add to result entries
- If entry is protected progress: add to result, increment protected_skipped
- Otherwise: add to result entries

Update `strategies/mod.rs` to declare and re-export `pub mod progress_collapse;`
Wire up `PruneStrategy::ProgressCollapse` arm in `strategy.rs` to call this function.

Tests for stale_reads:
1. Empty entries -> returns empty, zero savings
2. Single read of a file -> kept (not stale)
3. Two reads of same file path -> first removed, second (last) kept
4. Three reads of same file: reads 1 and 2 removed, read 3 kept
5. Reads of different files -> all kept (none are stale)
6. Protected stale read -> kept, protected_skipped incremented
7. Non-read user entries -> not affected
8. Mixed: user messages, reads, assistant entries -> only stale reads removed, order preserved
9. Read tool name variants: both "Read" and "read" detected (match existing diagnostics.rs logic)
</behavior>
<implementation>
Create `crates/assay-core/src/context/pruning/strategies/stale_reads.rs`:

```rust
pub fn stale_reads(
    entries: Vec<ParsedEntry>,
    _tier: PrescriptionTier,
    protected: &HashSet<usize>,
) -> StrategyResult
```

Logic (two-pass approach for "keep last read"):
1. **First pass:** Scan all entries to find the last occurrence of each file path read. Build a HashMap<String, usize> mapping file_path -> line_number of the last read.
2. **Second pass:** Iterate entries. For each User entry containing a Read/read tool_use:
   - Extract file_path from input.file_path
   - If this is NOT the last read of this path (line_number != last_read_map[path]) AND not protected:
     - Remove it (don't add to result), track savings
   - Otherwise: keep it
3. Non-read entries pass through unchanged.

Detection logic: Match the pattern from `diagnostics.rs` lines 126-139:
- User entry with message as array
- Block with type "tool_use" and name "Read" or "read"
- Extract file_path from input.file_path

Update `strategies/mod.rs` to declare `pub mod stale_reads;`
Wire up `PruneStrategy::StaleReads` arm in `strategy.rs`.
