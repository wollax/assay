```rust
/// Prune session bloat using composable strategies
#[command(after_long_help = "\
Examples:
  Dry-run with standard tier (default):
    assay context prune 3201041c-df85-4c91-a485-7b8c189f7636

  Dry-run with aggressive tier:
    assay context prune 3201041c --tier aggressive

  Run a single strategy:
    assay context prune 3201041c --strategy thinking-blocks

  Actually modify the session file:
    assay context prune 3201041c --execute

  List available backups for restore:
    assay context prune 3201041c --restore

  Output as JSON:
    assay context prune 3201041c --json")]
Prune {
    /// Session ID (required)
    session_id: String,

    /// Prescription tier: gentle, standard, aggressive
    #[arg(long, default_value = "standard")]
    tier: String,

    /// Run a single strategy instead of a prescription
    #[arg(long, conflicts_with = "tier")]
    strategy: Option<String>,

    /// Actually modify the session file (default is dry-run)
    #[arg(long)]
    execute: bool,

    /// List and restore from a previous backup
    #[arg(long, conflicts_with_all = ["tier", "strategy", "execute"])]
    restore: bool,

    /// Output as JSON
    #[arg(long)]
    json: bool,

    /// Plain output (no color, no Unicode)
    #[arg(long)]
    plain: bool,
},
```

<action>
Implement the `handle_context_prune` function:

```rust
fn handle_context_prune(
    session_id: &str,
    tier_str: &str,
    strategy_str: Option<&str>,
    execute: bool,
    restore: bool,
    json: bool,
    plain: bool,
) -> anyhow::Result<i32>
```

**Restore mode** (if `restore` is true):
1. Resolve session directory and session path using `assay_core::context::find_session_dir` and `resolve_session`
2. Determine backup directory: `.assay/backups/` under project root
3. Call `assay_core::context::pruning::backup::list_backups(backup_dir, session_id)`
4. If no backups, print "No backups found for session {session_id}" and return Ok(0)
5. If JSON: print backups as JSON array of paths
6. Otherwise: print numbered list of backups with timestamps, prompt user to use `--restore` with a specific backup path (per CONTEXT.md: no interactive restore, list only)

**Prune mode** (default):
1. Parse tier_str into PrescriptionTier. Bail if invalid.
2. Determine strategies:
   - If `strategy_str` is Some: parse into single PruneStrategy, bail if invalid
   - Otherwise: use `tier.strategies()` to get the prescription's strategy list
3. Resolve session path using find_session_dir + resolve_session
4. Determine backup_dir: `.assay/backups/` under project root (or fallback to session dir parent)
5. Call `assay_core::context::pruning::prune_session(session_path, strategies, tier, execute, Some(backup_dir))`
6. If JSON: serialize PruneReport and print
7. Otherwise: call `assay_core::context::pruning::report::format_dry_run_report(&report, !plain)` and print

**Error handling:**
- Invalid tier string -> bail with "unknown prescription tier: {tier_str}. Valid: gentle, standard, aggressive"
- Invalid strategy string -> bail with "unknown strategy: {strategy_str}. Valid: progress-collapse, stale-reads, ..."
- Session not found -> propagate assay_core error

**String-to-enum parsing:** Add `FromStr` or a `from_str` method on PrescriptionTier and PruneStrategy in the types crate, or do the parsing inline in the CLI handler. Prefer a simple match block in the handler to keep types crate clean:
```rust
let tier = match tier_str {
    "gentle" => PrescriptionTier::Gentle,
    "standard" => PrescriptionTier::Standard,
    "aggressive" => PrescriptionTier::Aggressive,
    _ => bail!("unknown prescription tier: {tier_str}"),
};
```
