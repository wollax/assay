<implementation>
**tool_output_trim.rs:**
```rust
/// Default line threshold for tool output trimming.
const TRIM_THRESHOLD_LINES: usize = 100;
/// Lines to keep from start of output.
const KEEP_HEAD_LINES: usize = 20;
/// Lines to keep from end of output.
const KEEP_TAIL_LINES: usize = 20;

pub fn tool_output_trim(
    entries: Vec<ParsedEntry>,
    _tier: PrescriptionTier,
    protected: &HashSet<usize>,
) -> StrategyResult
```

Logic for each User entry (not protected):
1. Check if message is an array of blocks
2. For each block with type "tool_result":
   - Get content as string (or to_string for Value)
   - Count lines; if > TRIM_THRESHOLD_LINES:
     - Take first KEEP_HEAD_LINES + "[...N lines truncated...]" + last KEEP_TAIL_LINES
     - Replace the content in the block
3. If any block was modified: reconstruct the User entry's message Value, call update_content()
4. Track bytes_saved as delta

This is a simple line-based heuristic as specified in CONTEXT.md ("initial: simpler line-based heuristics with extension point").

Update strategies/mod.rs, wire up in strategy.rs.

4. Three occurrences -> first two removed, last kept
5. Different system reminders -> each keeps its own last occurrence
6. System reminder in assistant entry text -> detected and deduped

8. Protected entry with duplicate system reminder -> kept, protected_skipped incremented
9. bytes_saved reflects removal of duplicate reminder lines
</behavior>
<implementation>
**system_reminder_dedup.rs:**

Uses the same `<system-reminder>` regex pattern from diagnostics.rs.

```rust
pub fn system_reminder_dedup(
    entries: Vec<ParsedEntry>,
    _tier: PrescriptionTier,
    protected: &HashSet<usize>,
) -> StrategyResult
```

Two-pass approach:
1. **First pass (reverse scan):** Iterate entries in reverse to find the last occurrence of each unique system reminder content. Build a HashSet<usize> of line numbers to KEEP (the last occurrence of each unique reminder).
2. **Second pass (forward):** Iterate entries forward. For entries containing system reminders:
   - If line_number is in the keep set -> keep the entry
   - If protected -> keep and count protected_skipped
   - Otherwise -> remove (it's a duplicate), track savings

System reminder detection: Check text content for `<system-reminder>` tag (using regex_lite, matching the pattern in diagnostics.rs). For dedup purposes, extract the content between `<system-reminder>` and `</system-reminder>` as the dedup key, or use the full text block content as the key if extraction is too complex.

Simpler approach: Since system reminders are typically the entire text block content, use the full text block as the dedup key. Two entries with the same text block content containing `<system-reminder>` are duplicates; keep the last.

Update strategies/mod.rs, wire up in strategy.rs.
