<behavior>
Tests for execute_pipeline:
1. Empty strategy list -> entries unchanged, no strategy_results
2. Single strategy -> entries transformed, one StrategyResult recorded
3. Two strategies in sequence -> second operates on output of first, both results recorded
4. original_size and final_size calculated from raw_bytes sums
5. Protection set passed to each strategy
6. Strategy order matches input order (not re-sorted)

Tests for write_session:
1. Writes entries' raw_line values as JSONL (one per line)
2. Atomic write: file appears complete (no partial writes observable)
3. Handles empty entries -> creates empty file

Tests for prune_session (integration-level):
1. Dry-run (execute=false) -> returns PruneReport with executed=false, file unchanged
2. Execute mode -> returns PruneReport with executed=true, file is modified
3. Parse -> protect -> pipeline -> report flow works end-to-end
</behavior>
<implementation>
In `crates/assay-core/src/context/pruning/mod.rs`, implement:

1. **execute_pipeline:**
```rust
pub fn execute_pipeline(
    entries: Vec<ParsedEntry>,
    strategies: &[PruneStrategy],
    tier: PrescriptionTier,
    protected_lines: &HashSet<usize>,
) -> (Vec<ParsedEntry>, Vec<(PruneStrategy, StrategyResult)>)
```
- Track original size from initial entries
- Apply each strategy sequentially, collecting StrategyResult
- Use a pattern where StrategyResult.entries is moved into the next strategy (avoid clone)
- Return final entries and per-strategy results

2. **write_session:**
```rust
pub fn write_session(entries: &[ParsedEntry], target: &Path) -> crate::Result<()>
```
- Use tempfile::NamedTempFile::new_in(target.parent())
- Write each entry.raw_line as a line
- persist() to target path

3. **prune_session (top-level API):**
```rust
pub fn prune_session(
    session_path: &Path,
    strategies: &[PruneStrategy],
    tier: PrescriptionTier,
    execute: bool,
    backup_dir: Option<&Path>,
) -> crate::Result<PruneReport>
```
- Parse session (gets entries with raw_line)
- Build protection set
- Execute pipeline
- Build PruneReport from results
- If execute: backup session, then write_session
- Return PruneReport

Add appropriate re-exports from the module.
</implementation>

const DEFAULT_RETENTION_LIMIT: usize = 5;
