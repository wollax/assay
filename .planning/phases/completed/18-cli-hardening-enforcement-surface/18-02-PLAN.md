  <name>Task 1: Add warned counter and enforcement-aware streaming</name>
  <files>crates/assay-cli/src/main.rs</files>
  <action>
1. Add `warned: usize` field to `StreamCounters`:
   ```rust
   struct StreamCounters {
       passed: usize,
       failed: usize,
       warned: usize,
       skipped: usize,
   }
   ```
   Update all construction sites to include `warned: 0`.

2. Add a `format_warn()` function parallel to `format_pass()` and `format_fail()`:
   ```rust
   fn format_warn(color: bool) -> &'static str {
       if color { "\x1b[33mWARN\x1b[0m" } else { "WARN" }
   }
   ```

3. Change `stream_criterion()` signature to also accept `gate_section: Option<&assay_types::GateSection>`:
   ```rust
   fn stream_criterion(
       criterion: &assay_types::Criterion,
       working_dir: &std::path::Path,
       cfg: &StreamConfig,
       counters: &mut StreamCounters,
       gate_section: Option<&assay_types::GateSection>,
   )
   ```

4. Inside `stream_criterion()`, resolve enforcement for the criterion:
   ```rust
   let enforcement = assay_core::gate::resolve_enforcement(criterion.enforcement, gate_section);
   let is_advisory = enforcement == assay_types::Enforcement::Advisory;
   ```

5. For **passing** criteria that are advisory, prepend `[advisory]` label:
   ```rust
   // In the Ok(result) branch where result.passed is true:
   if is_advisory {
       eprintln!("{cr}  {label} [advisory] {} ... {}", criterion.name, format_pass(cfg.color));
   } else {
       eprintln!("{cr}  {label} {} ... {}", criterion.name, format_pass(cfg.color));
   }
   ```

6. For **failing** criteria, use enforcement to decide label and counter:
   - Advisory failure: increment `counters.warned` (NOT `counters.failed`), display yellow `WARN`
   - Required failure: increment `counters.failed`, display red `FAILED`
   ```rust
   // In the Ok(result) branch where result.passed is false:
   if is_advisory {
       counters.warned += 1;
       let label_str = format_warn(cfg.color);
       eprintln!("{cr}  {label} [advisory] {} ... {}", criterion.name, label_str);
   } else {
       counters.failed += 1;
       eprintln!("{cr}  {label} {} ... {}", criterion.name, format_fail(cfg.color));
   }
   ```
   Same split in the `Err(err)` branch.

7. Update all call sites of `stream_criterion()` to pass `gate_section`. There are two call sites:
   - In `handle_gate_run()`: already has `gate_section` in scope.
   - In `handle_gate_run_all()`: already has `gate_section` in scope.

8. Remove the post-hoc `resolve_enforcement` checks in `handle_gate_run()` and `handle_gate_run_all()` that currently track `has_required_failure` by comparing `counters.failed` before/after. Instead, determine required failure from `counters.failed > 0` (since `counters.failed` now only counts required failures).

   In `handle_gate_run()`, replace:
   ```rust
   let mut has_required_failure = false;
   for criterion in &criteria {
       let before_failed = counters.failed;
       stream_criterion(...);
       if counters.failed > before_failed {
           let enforcement = ...;
           if enforcement == Required { has_required_failure = true; }
       }
   }
   // ...
   if has_required_failure { return Ok(1); }
   ```
   With:
   ```rust
   for criterion in &criteria {
       stream_criterion(criterion, &working_dir, &cfg, &mut counters, gate_section);
   }
   // ...
   if counters.failed > 0 { return Ok(1); }
   ```

   Same simplification in `handle_gate_run_all()` — `has_required_failure` becomes `counters.failed > 0` at the end, or track per-spec `before_failed` delta for the `--all` case.
