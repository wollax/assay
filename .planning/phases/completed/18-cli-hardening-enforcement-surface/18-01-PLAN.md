<task type="auto">
  <name>Task 2: Migrate main() to run() pattern with anyhow::Result<i32></name>
  <files>crates/assay-cli/src/main.rs</files>
  <action>
1. Add `use anyhow::{bail, Context};` to imports.

2. Create `async fn run() -> anyhow::Result<i32>` that contains the current body of `main()`. Transform the dispatch logic:
   - Replace `Cli::parse()` with `Cli::try_parse().unwrap_or_else(|e| e.exit())` to preserve clap's native error formatting (exit code 2).
   - The `None` arm (bare invocation): if `assay_dir(&root).is_dir()`, call `show_status(&root)?` and return `Ok(0)`. Otherwise, print help + hint to stderr and return `Ok(1)`.
   - All `Some(Command::...)` arms: call the corresponding handler (which now returns `anyhow::Result<i32>`) and propagate with `?` or return the exit code.

3. Rewrite `main()` to the catch-at-top pattern:
   ```rust
   #[tokio::main]
   async fn main() {
       let code = match run().await {
           Ok(code) => code,
           Err(e) => {
               eprintln!("Error: {e:#}");
               1
           }
       };
       std::process::exit(code);
   }
   ```

4. Convert `project_root()` from exit-on-error to returning `anyhow::Result<PathBuf>`:
   ```rust
   fn project_root() -> anyhow::Result<std::path::PathBuf> {
       std::env::current_dir().context("could not determine current directory")
   }
   ```

5. Convert `load_gate_context()` to return `anyhow::Result<(PathBuf, Config, PathBuf, Option<u64>)>`:
   - Replace the `process::exit(1)` with `?` on config load.

6. Convert ALL handler functions to return `anyhow::Result<i32>`:
   - `handle_spec_show` → `anyhow::Result<i32>` — config load errors use `?`, spec-not-found uses `bail!("spec '{name}' not found in {}", config.specs_dir)`, serialization errors use `?`. Return `Ok(0)`.
   - `handle_spec_list` → `anyhow::Result<i32>` — same pattern.
   - `handle_spec_new` → `anyhow::Result<i32>` — same pattern.
   - `handle_gate_run` → `anyhow::Result<i32>` — return `Ok(1)` for required failures, `Ok(0)` otherwise. Replace `process::exit(1)` on spec-not-found/config-load with `?`/`bail!`.
   - `handle_gate_run_all` → `anyhow::Result<i32>` — same. Return `Ok(1)` if `has_required_failure`.
   - `handle_gate_history` → `anyhow::Result<i32>` — same pattern.
   - `handle_gate_history_detail` → `anyhow::Result<i32>` — same pattern.
   - `show_status` → change from `Result<(), String>` to `anyhow::Result<()>`.

7. For each handler, systematically replace every `process::exit(1)` with the appropriate mechanism:
   - Config/IO errors: use `?` with `.context("...")` where helpful
   - Spec not found: `bail!("spec '{}' not found in {}", name, dir)`
   - Serialization: `.context("failed to serialize ...")?`
   - Gate failures (business logic): `return Ok(1)`

8. After migration, there must be exactly ONE `process::exit()` call in the entire file — inside `main()`.
