<task type="auto">
  <name>Task 2: Define WorkSession types with state machine and JSON round-trip tests</name>
  <files>crates/assay-types/src/work_session.rs, crates/assay-types/src/lib.rs, crates/assay-core/src/error.rs</files>
  <action>
  **Types to create in `crates/assay-types/src/work_session.rs`:**

  1. **`SessionPhase`** enum — `#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, Serialize, Deserialize, JsonSchema)]` with `#[serde(rename_all = "snake_case")]`. Variants: `Created`, `AgentRunning`, `GateEvaluated`, `Completed`, `Abandoned`.

     Implement two methods:
     - `pub fn can_transition_to(&self, next: SessionPhase) -> bool` — allows Created->AgentRunning, AgentRunning->GateEvaluated, GateEvaluated->Completed, and ANY state->Abandoned. All other transitions (including from terminal states) return false.
     - `pub fn is_terminal(&self) -> bool` — true for Completed and Abandoned only.

  2. **`PhaseTransition`** struct — `#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize, JsonSchema)]`. Fields: `from: SessionPhase`, `to: SessionPhase`, `timestamp: DateTime<Utc>`, `trigger: String`, `notes: Option<String>` (with `#[serde(default, skip_serializing_if = "Option::is_none")]`).

  3. **`AgentInvocation`** struct — `#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize, JsonSchema)]`. Fields: `spec_name: String`, `command: String`, `model: Option<String>` (with `#[serde(default, skip_serializing_if = "Option::is_none")]`).

  4. **`WorkSession`** struct — `#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize, JsonSchema)]`. NO `deny_unknown_fields` (mutable document, evolves in Phases 41/42). Fields:
     - `id: String` (ULID stored as string for schemars compatibility)
     - `spec_name: String`
     - `worktree_path: PathBuf`
     - `phase: SessionPhase`
     - `transitions: Vec<PhaseTransition>`
     - `agent: AgentInvocation`
     - `gate_runs: Vec<String>` with `#[serde(default, skip_serializing_if = "Vec::is_empty")]`
     - `assay_version: String`

  5. Register all four types with the schema registry using `inventory::submit!` (same pattern as other types in lib.rs).

  **Module wiring in `crates/assay-types/src/lib.rs`:**
  - Add `pub mod work_session;`
  - Add re-exports: `pub use work_session::{WorkSession, SessionPhase, PhaseTransition, AgentInvocation};`

  **Error variants in `crates/assay-core/src/error.rs`:**
  Add two new variants to `AssayError` (before the closing `}` of the enum, after the last existing variant):

  ```rust
  /// WorkSession transition error (invalid phase transition).
  #[error("work session `{session_id}` transition error: {message}")]
  WorkSessionTransition {
      /// The session ID involved.
      session_id: String,
      /// Description of the invalid transition.
      message: String,
  },

  /// WorkSession not found on disk.
  #[error("work session `{session_id}` not found")]
  WorkSessionNotFound {
      /// The session ID that was looked up.
      session_id: String,
  },
  ```

  **Tests** (in `work_session.rs`, `#[cfg(test)] mod tests`):

  TDD sequence — write these tests RED first, then implement:

  1. `session_phase_serializes_as_snake_case` — verify `SessionPhase::AgentRunning` serializes to `"agent_running"` and `GateEvaluated` to `"gate_evaluated"`.

  2. `can_transition_happy_path` — verify all valid forward transitions: Created->AgentRunning, AgentRunning->GateEvaluated, GateEvaluated->Completed.

  3. `can_transition_to_abandoned_from_any` — verify Created->Abandoned, AgentRunning->Abandoned, GateEvaluated->Abandoned all return true.

  4. `cannot_transition_from_terminal` — verify Completed and Abandoned cannot transition to any state (including Abandoned->Abandoned).

  5. `cannot_skip_phases` — verify Created->GateEvaluated, Created->Completed, AgentRunning->Completed all return false.

  6. `is_terminal` — verify only Completed and Abandoned return true.

  7. `work_session_json_round_trip` — create a full WorkSession with transitions and gate_runs, serialize to JSON, deserialize back, assert equality.

  8. `work_session_tolerates_unknown_fields` — deserialize a JSON string with an extra unknown field, verify it succeeds (no deny_unknown_fields).

  9. `phase_transition_optional_notes_omitted` — serialize a PhaseTransition with `notes: None`, verify the JSON output does not contain the "notes" key.

  10. `gate_runs_omitted_when_empty` — serialize a WorkSession with empty gate_runs vec, verify the JSON output does not contain the "gate_runs" key.
