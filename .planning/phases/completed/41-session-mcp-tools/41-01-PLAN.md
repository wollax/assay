<objective>
Add four session MCP tools (`session_create`, `session_get`, `session_update`, `session_list`) to the AssayServer, wiring them to the existing persistence layer in `assay-core::work_session`.
</objective>

<context>
Phase 40 delivered `WorkSession` types (assay-types) and persistence functions (assay-core):
- `create_work_session(spec_name, worktree_path, agent_command, agent_model) -> WorkSession`
- `transition_session(session, next, trigger, notes) -> Result<()>`
- `save_session(assay_dir, session) -> Result<PathBuf>`
- `load_session(assay_dir, session_id) -> Result<WorkSession>`
- `list_sessions(assay_dir) -> Result<Vec<String>>` (returns IDs only)

Types live in `assay_types::work_session::{WorkSession, SessionPhase, PhaseTransition, AgentInvocation}`.

The MCP server pattern is well-established in `server.rs`:
- Param structs: `#[derive(Deserialize, JsonSchema)]` with `#[schemars(description = "...")]` on each field
- Response structs: `#[derive(Serialize)]` with `warnings: Vec<String>` field using `#[serde(default, skip_serializing_if = "Vec::is_empty")]`
- Tool methods: `#[tool(description = "...")]` on `async fn` taking `Parameters<T>`, returning `Result<CallToolResult, McpError>`
- Boilerplate: `resolve_cwd()` + `load_config()` at start, `domain_error(&e)` for errors
- `assay_dir` = `cwd.join(".assay")` (inline pattern, not extracted)
- Param structs exported under `#[cfg(any(test, feature = "testing"))]` in `lib.rs`

A `session_get` tool is included beyond the 3 requirements because `list_sessions` returns IDs only. Without `session_get`, agents must use `session_list` (which loads every session for filtering) even when they know the ID. This is token-wasteful. `session_get` provides O(1) lookup by ID.

Spec validation on `session_create` ensures the spec actually exists before creating a session, consistent with `gate_run` behavior.
</context>

### Task 3: Implement `session_create` tool method

Add the tool method inside the `#[tool_router] impl AssayServer` block:

```rust
#[tool(
    description = "Create a new work session for a spec. Returns the session ID and initial state. \
        The session starts in 'created' phase. Use session_update to advance through phases \
        (agent_running → gate_evaluated → completed) or abandon. Validates that the spec exists."
)]
pub async fn session_create(
    &self,
    params: Parameters<SessionCreateParams>,
) -> Result<CallToolResult, McpError> {
    let cwd = resolve_cwd()?;
    let config = match load_config(&cwd) {
        Ok(c) => c,
        Err(err_result) => return Ok(err_result),
    };
    let mut warnings = Vec::new();

    // Validate spec exists (consistent with gate_run)
    if let Err(err_result) = load_spec_entry_mcp(&cwd, &config, &params.0.spec_name) {
        return Ok(err_result);
    }

    let assay_dir = cwd.join(".assay");
    let session = assay_core::work_session::create_work_session(
        &params.0.spec_name,
        PathBuf::from(&params.0.worktree_path),
        &params.0.agent_command,
        params.0.agent_model.as_deref(),
    );

    if let Err(e) = assay_core::work_session::save_session(&assay_dir, &session) {
        return Ok(domain_error(&e));
    }

    let response = SessionCreateResponse {
        session_id: session.id,
        spec_name: session.spec_name,
        phase: session.phase.to_string(),
        created_at: session.created_at.to_rfc3339(),
        warnings,
    };

    let json = serde_json::to_string(&response)
        .map_err(|e| McpError::internal_error(format!("serialization failed: {e}"), None))?;
    Ok(CallToolResult::success(vec![Content::text(json)]))
}
```

### Task 5: Implement `session_update` tool method

```rust
#[tool(
    description = "Update a work session: transition phase and/or link gate run IDs. \
        Valid transitions: created→agent_running, agent_running→gate_evaluated, \
        gate_evaluated→completed, any non-terminal→abandoned. \
        Invalid transitions are rejected with a clear error message."
)]
pub async fn session_update(
    &self,
    params: Parameters<SessionUpdateParams>,
) -> Result<CallToolResult, McpError> {
    let cwd = resolve_cwd()?;
    let assay_dir = cwd.join(".assay");
    let mut warnings = Vec::new();

    let mut session = match assay_core::work_session::load_session(&assay_dir, &params.0.session_id) {
        Ok(s) => s,
        Err(e) => return Ok(domain_error(&e)),
    };

    let previous_phase = session.phase;

    if let Err(e) = assay_core::work_session::transition_session(
        &mut session,
        params.0.phase,
        &params.0.trigger,
        params.0.notes.as_deref(),
    ) {
        return Ok(domain_error(&e));
    }

    // Append gate run IDs
    for id in &params.0.gate_run_ids {
        if !session.gate_runs.contains(id) {
            session.gate_runs.push(id.clone());
        }
    }

    if let Err(e) = assay_core::work_session::save_session(&assay_dir, &session) {
        return Ok(domain_error(&e));
    }

    let response = SessionUpdateResponse {
        session_id: session.id,
        previous_phase: previous_phase.to_string(),
        current_phase: session.phase.to_string(),
        gate_runs_count: session.gate_runs.len(),
        warnings,
    };

    let json = serde_json::to_string(&response)
        .map_err(|e| McpError::internal_error(format!("serialization failed: {e}"), None))?;
    Ok(CallToolResult::success(vec![Content::text(json)]))
}
```

### Task 6: Implement `session_list` tool method

Since `list_sessions` returns IDs only, this tool must load each session to apply filters. This is acceptable for the expected session count (tens, not thousands).

```rust
#[tool(
    description = "List work sessions with optional filters. Returns summary entries \
        (id, spec_name, phase, created_at, gate_runs_count). \
        Filter by spec_name (exact match) and/or status (phase). \
        Results are ordered chronologically (oldest first, ULID order)."
)]
pub async fn session_list(
    &self,
    params: Parameters<SessionListParams>,
) -> Result<CallToolResult, McpError> {
    let cwd = resolve_cwd()?;
    let assay_dir = cwd.join(".assay");
    let mut warnings = Vec::new();

    let ids = match assay_core::work_session::list_sessions(&assay_dir) {
        Ok(ids) => ids,
        Err(e) => return Ok(domain_error(&e)),
    };

    let total = ids.len();
    let limit = params.0.limit.unwrap_or(20).min(100);

    let mut sessions = Vec::new();
    for id in &ids {
        let session = match assay_core::work_session::load_session(&assay_dir, id) {
            Ok(s) => s,
            Err(e) => {
                warnings.push(format!("skipping session {id}: {e}"));
                continue;
            }
        };

        // Apply filters
        if let Some(ref spec_filter) = params.0.spec_name {
            if session.spec_name != *spec_filter {
                continue;
            }
        }
        if let Some(status_filter) = params.0.status {
            if session.phase != status_filter {
                continue;
            }
        }

        sessions.push(SessionListEntry {
            id: session.id,
            spec_name: session.spec_name,
            phase: session.phase.to_string(),
            created_at: session.created_at.to_rfc3339(),
            gate_runs_count: session.gate_runs.len(),
        });

        if sessions.len() >= limit {
            break;
        }
    }

    let response = SessionListResponse {
        total,
        sessions,
        warnings,
    };

    let json = serde_json::to_string(&response)
        .map_err(|e| McpError::internal_error(format!("serialization failed: {e}"), None))?;
    Ok(CallToolResult::success(vec![Content::text(json)]))
}
```
