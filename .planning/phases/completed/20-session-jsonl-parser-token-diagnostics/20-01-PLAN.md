  **Dependency promotion:** In `crates/assay-types/Cargo.toml`, move `serde_json` from `[dev-dependencies]` to `[dependencies]` (as `serde_json.workspace = true`). The new context types use `serde_json::Value` in non-dev code (e.g., `ProgressEntry.data`, `FileHistorySnapshot(serde_json::Value)`), so it must be a full dependency.

  Create `crates/assay-types/src/context.rs` with the following types. Use existing crate patterns (derive Debug, Clone, Serialize, Deserialize, JsonSchema where appropriate).

  **JSONL Entry Types (for parser input):**

  ```rust
  #[derive(Debug, Clone, Serialize, Deserialize)]
  #[serde(tag = "type", rename_all = "kebab-case")]
  pub enum SessionEntry { ... }
  ```

  Variants:
  - `User(UserEntry)` ŌĆö user messages and tool results
  - `Assistant(AssistantEntry)` ŌĆö model responses with content blocks and optional usage
  - `Progress(ProgressEntry)` ŌĆö hook/agent/bash progress ticks (use `serde_json::Value` for `data`)
  - `System(SystemEntry)` ŌĆö system entries like compact_boundary, stop_hook_summary
  - `FileHistorySnapshot(serde_json::Value)` ŌĆö not needed for diagnostics, capture raw
  - `QueueOperation(serde_json::Value)` ŌĆö not needed for diagnostics
  - `PrLink(serde_json::Value)` ŌĆö not needed for diagnostics
  - `#[serde(other)] Unknown` ŌĆö defensive catch-all for future entry types

  Note: `rename_all = "kebab-case"` handles `file-history-snapshot`, `queue-operation`, `pr-link` automatically.

  **Common fields struct** (shared top-level fields present on all typed entries):
  ```rust
  #[derive(Debug, Clone, Serialize, Deserialize)]
  pub struct EntryMetadata {
      pub uuid: String,
      pub timestamp: String,
      #[serde(rename = "sessionId")]
      pub session_id: String,
      #[serde(rename = "parentUuid")]
      pub parent_uuid: Option<String>,
      #[serde(rename = "isSidechain", default)]
      pub is_sidechain: bool,
      #[serde(default)]
      pub cwd: Option<String>,
      #[serde(default)]
      pub version: Option<String>,
  }
  ```

  Note: `EntryMetadata` cannot be flattened into `SessionEntry` directly because `#[serde(tag)]` and `#[serde(flatten)]` don't compose well. Instead, the parser will extract metadata separately. Design the entry variant structs to include the metadata fields they need.

  Actually, the simpler approach: make each variant struct (`UserEntry`, `AssistantEntry`, `ProgressEntry`, `SystemEntry`) contain the common fields directly plus their specific fields. Use `#[serde(flatten)]` on an `EntryMetadata` field inside each variant struct.

  Wait ŌĆö `#[serde(tag)]` on the enum + `#[serde(flatten)]` inside variants works fine (tag is consumed from the map, flatten reads the rest). Use this pattern:

  ```rust

  pub struct UserEntry {
      #[serde(flatten)]
      pub meta: EntryMetadata,
      #[serde(default)]
      pub message: Option<serde_json::Value>,
  }
  ```

  **Token/Usage types:**
  ```rust
  #[derive(Debug, Clone, Default, Serialize, Deserialize, JsonSchema)]
  pub struct UsageData {
      #[serde(default)]
      pub input_tokens: u64,
      #[serde(default)]
      pub output_tokens: u64,
      #[serde(default)]
      pub cache_creation_input_tokens: u64,
      #[serde(default)]
      pub cache_read_input_tokens: u64,
  }

  impl UsageData {
      /// Total tokens in the context window for this API call.
      /// = input_tokens + cache_creation_input_tokens + cache_read_input_tokens
      pub fn context_tokens(&self) -> u64 {
          self.input_tokens + self.cache_creation_input_tokens + self.cache_read_input_tokens
      }
  }
  ```
