# Plan 19-01: Tighten cargo-deny Policies

## Goal

Switch cargo-deny `multiple-versions` from `warn` to `deny` with appropriate skip entries for known transitive duplicates, and switch `sources` from `warn` to `deny`.

## Tasks

<task id="1">
<files>deny.toml</files>
<action>
Update deny.toml:

1. In `[bans]`, change `multiple-versions = "warn"` to `multiple-versions = "deny"`
2. Add `skip` and `skip-tree` entries for known transitive duplicates. Run `cargo deny check bans 2>&1` first to get the current list of duplicates. Based on research, expect these duplicates (verify with actual output):
   - crossterm (0.28 vs 0.29) — ratatui transitively uses 0.29 via ratatui-crossterm
   - getrandom (0.3 vs 0.4) — ahash in jsonschema uses old version
   - linux-raw-sys — follows rustix version split
   - rustix — crossterm 0.28 uses polling 3.x which needs old rustix
   - windows-sys family — foundational Windows crate version fragmentation
   Use `skip` for individual crates and `skip-tree` for the windows-sys family. Each entry must have a `reason` string.
3. In `[sources]`, change `unknown-registry = "warn"` to `unknown-registry = "deny"` and `unknown-git = "warn"` to `unknown-git = "deny"`
4. Run `cargo deny check` to verify all checks pass (advisories, licenses, bans, sources)
5. Run `just deny` to confirm the just target also passes
</action>
<verify>
- `cargo deny check bans` exits 0
- `cargo deny check sources` exits 0
- `cargo deny check` (all checks) exits 0
- `just deny` exits 0
</verify>
<done>deny.toml has multiple-versions=deny with skip entries, and sources set to deny. All cargo-deny checks pass.</done>
</task>
