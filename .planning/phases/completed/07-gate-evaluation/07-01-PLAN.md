<task type="auto">
  <name>Task 2: Implement gate evaluation engine</name>
  <files>

  <action>
Replace the stub `crates/assay-core/src/gate/mod.rs` with the full gate evaluation module. Follow the architecture patterns from 07-RESEARCH.md closely. The module should contain:

**Constants:**
- `MAX_OUTPUT_BYTES: usize = 65_536` (64 KB hardcoded truncation limit)
- `POLL_INTERVAL_MS: u64 = 50` (try_wait polling interval)
- `MIN_TIMEOUT_SECS: u64 = 1` (minimum timeout floor)

**Public types (defined in this module, not in assay-types — these are computed summaries):**
- `GateRunSummary { spec_name: String, results: Vec<CriterionResult>, passed: usize, failed: usize, skipped: usize, total_duration_ms: u64 }` — derive `Debug, Clone, Serialize`
- `CriterionResult { criterion_name: String, result: Option<GateResult> }` — derive `Debug, Clone, Serialize`. `result` is `None` for skipped (descriptive-only) criteria.

**Public functions:**

1. `evaluate(criterion: &Criterion, working_dir: &Path, timeout: Duration) -> Result<GateResult>` — Main entry point. Derives `GateKind` from criterion fields: if `cmd` is Some, it's `GateKind::Command`; if `cmd` is None, it's `GateKind::AlwaysPass`. Note: `FileExists` is not derived from Criterion — it will be used when criteria reference file checks in future phases. For now, evaluate always maps cmd -> Command, no cmd -> AlwaysPass. Dispatches to private helpers.

2. `evaluate_all(spec: &Spec, working_dir: &Path, cli_timeout: Option<u64>, config_timeout: Option<u64>) -> GateRunSummary` — Evaluates all criteria in a spec sequentially. Skips criteria without `cmd` (records as skipped in summary). Uses `resolve_timeout()` for each criterion. Collects results into GateRunSummary. Does NOT return Result — individual criterion failures are captured in GateResult, not propagated as errors. If `evaluate()` returns an Err (spawn failure), capture it as a failed CriterionResult with error details in stderr.

3. `resolve_timeout(cli_timeout: Option<u64>, criterion_timeout: Option<u64>, config_timeout: Option<u64>) -> Duration` — Three-tier precedence: cli > criterion > config > 300s default. Enforce minimum 1 second floor.

**Private functions:**

4. `evaluate_command(cmd: &str, working_dir: &Path, timeout: Duration) -> Result<GateResult>` — Spawn `sh -c <cmd>` with `Stdio::piped()`, `current_dir(working_dir)`, and `process_group(0)` (Unix, `#[cfg(unix)]`). Take stdout/stderr handles, spawn reader threads with `read_to_end`. Poll `try_wait` in a loop with 50ms sleep for timeout. On timeout: `child.kill()` then `child.wait()` to reap. Join reader threads. Convert bytes with `String::from_utf8_lossy()`. Apply `truncate_output()`. Build GateResult with appropriate passed/exit_code/timeout info. On timeout, append `[timed out after Ns]` to stderr, set `exit_code: None`, `passed: false`.

5. `evaluate_always_pass() -> Result<GateResult>` — Return immediately with `passed: true`, `GateKind::AlwaysPass`, empty stdout/stderr, no exit_code, 0 duration_ms.

6. `evaluate_file_exists(path: &str, working_dir: &Path) -> Result<GateResult>` — Resolve path relative to working_dir, check `Path::exists()`. Return GateResult with `passed: exists`, `GateKind::FileExists { path }`, stderr with "file not found: {full_path}" if missing.

7. `truncate_output(output: &str, max_bytes: usize) -> (String, bool)` — Returns `(possibly_truncated_string, was_truncated)`. If within budget, return as-is. Otherwise, keep the tail (errors appear at end), prepend `[truncated, showing last {max_bytes} bytes]\n`. Use `str::ceil_char_boundary()` for safe UTF-8 slicing.

8. Helper to build the final GateResult with truncation metadata: after getting stdout and stderr from truncate_output, set `truncated: true` if either was truncated, and `original_bytes` to the sum of original byte counts if truncated.

**Tests (comprehensive, in `#[cfg(test)] mod tests`):**

- `evaluate_echo_hello`: Run `echo hello`, assert passed=true, stdout contains "hello", exit_code=Some(0), duration_ms > 0.
- `evaluate_failing_command`: Run `sh -c "echo fail >&2 && exit 1"`, assert passed=false, stderr contains "fail", exit_code=Some(1).
- `evaluate_timeout`: Run `sleep 10` with 1s timeout, assert passed=false, exit_code=None, stderr contains "timed out".
- `evaluate_always_pass_criterion`: Criterion with no cmd, assert passed=true, kind is AlwaysPass.
- `evaluate_file_exists_present`: Create a temp file, evaluate FileExists, assert passed=true.
- `evaluate_file_exists_missing`: Evaluate FileExists on nonexistent path, assert passed=false, stderr contains "file not found".
- `evaluate_working_dir_is_respected`: Run `pwd`, assert stdout contains the working_dir path.
- `truncate_output_within_budget`: String under limit returns unchanged, not truncated.
- `truncate_output_over_budget`: Large string is tail-truncated, returns truncated=true.
- `resolve_timeout_cli_wins`: CLI timeout overrides all others.
- `resolve_timeout_criterion_wins_over_config`: Per-criterion beats config.
- `resolve_timeout_config_used`: Config used when no CLI or criterion timeout.
- `resolve_timeout_default_300s`: No overrides returns 300s.
- `resolve_timeout_minimum_floor`: Timeout of 0 becomes 1s.
- `evaluate_all_mixed_criteria`: Spec with executable + descriptive criteria, verify passed/failed/skipped counts.
- `evaluate_all_captures_spawn_failure`: Criterion with impossible command (e.g., nonexistent binary path), verify it's captured as failure not panic.

Use `tempfile::tempdir()` for working directories. Mark the timeout test with `#[cfg_attr(not(unix), ignore)]` since process_group is Unix-only. The timeout test will take ~1-2 seconds — acceptable for correctness.

**Important constraints:**
- `evaluate_file_exists` is a standalone function that can be called directly, not derived from Criterion. It's for GateKind::FileExists. The `evaluate()` function's Criterion -> GateKind mapping only produces Command or AlwaysPass. FileExists is used directly via `evaluate_file_exists()` for now (future phases will add file-check criteria).
- NO async code anywhere in this module. All sync per GATE-08.
- The `#[cfg(unix)]` on `process_group(0)` means Windows builds compile but don't get process group kill (acceptable for v0.1.0).
- Do NOT use `Command::output()` — use `spawn()` + reader threads per research.
- Do NOT add any external dependencies beyond what's already wired.
