1. Add `Guard` variant to the `ContextCommand` enum:
```rust
/// Background context protection daemon
#[command(after_long_help = "\
Examples:
  Start the guard daemon:
    assay context guard start

  Start with a specific session:
    assay context guard start --session /path/to/session.jsonl

  Check daemon status:
    assay context guard status

  Stop the daemon:
    assay context guard stop

  View daemon logs:
    assay context guard logs
    assay context guard logs --level warn")]
Guard {
    #[command(subcommand)]
    command: GuardCommand,
},
```

2. Add `GuardCommand` enum:
```rust
#[derive(Subcommand)]
enum GuardCommand {
    /// Start the guard daemon (runs in foreground)
    Start {
        /// Session file path (auto-discovers if omitted)
        #[arg(long)]
        session: Option<String>,
    },
    /// Stop the running guard daemon
    Stop,
    /// Show guard daemon status
    Status,
    /// View guard daemon logs
    Logs {
        /// Filter by log level (trace, debug, info, warn, error)
        #[arg(long, default_value = "info")]
        level: String,
    },
}
```

3. Add match arm in the main dispatch:
```rust
ContextCommand::Guard { command } => match command {
    GuardCommand::Start { session } => handle_guard_start(session.as_deref()),
    GuardCommand::Stop => handle_guard_stop(),
    GuardCommand::Status => handle_guard_status(),
    GuardCommand::Logs { level } => handle_guard_logs(&level),
},
```

4. Implement `handle_guard_start(session: Option<&str>) -> anyhow::Result<i32>`:
   ```
   a. Determine assay_dir: current directory + ".assay"
   b. Load config from .assay/config.toml. Extract guard config (use default if not configured).
   c. Determine session path:
      - If session arg provided: use it directly
      - Otherwise: auto-discover via assay_core::context::find_session_dir + resolve_session
   d. Set up guard log directory: .assay/guard/
   e. Set up tracing-appender: file appender to .assay/guard/guard.log
      Use tracing_subscriber with fmt layer writing to file + stderr summary layer
   f. Print to stderr: "[guard] Starting — watching {session_path}"
   g. Print to stderr: "[guard] Soft: {soft}%, Hard: {hard}%, Poll: {interval}s"
   h. Build tokio runtime and block on start_guard(session_path, assay_dir, config)
   i. Match result:
      - Ok(()) => print "[guard] Stopped cleanly.", return Ok(0)
      - Err(GuardCircuitBreakerTripped { .. }) => print error, return Ok(2)
      - Err(e) => return Err(e.into())
   ```

5. Implement `handle_guard_stop() -> anyhow::Result<i32>`:
   ```
   a. Determine assay_dir
   b. Call assay_core::guard::stop_guard(&assay_dir)
   c. Print "[guard] Daemon stopped."
   d. Return Ok(0)
   e. If GuardNotRunning error: print "Guard daemon is not running.", return Ok(1)
   ```

6. Implement `handle_guard_status() -> anyhow::Result<i32>`:
   ```
   a. Determine assay_dir
   b. Call assay_core::guard::guard_status(&assay_dir)
   c. Match:
      - Running { pid } => println!("Guard daemon running (PID {pid})"), return Ok(0)
      - Stopped => println!("Guard daemon is not running"), return Ok(1)
   ```

7. Implement `handle_guard_logs(level: &str) -> anyhow::Result<i32>`:
   ```
   a. Determine log file path: .assay/guard/guard.log
   b. If file doesn't exist: print "No guard logs found.", return Ok(1)
   c. Read file contents
   d. Filter lines by level (case-insensitive match on level string in log lines)
   e. Print filtered lines to stdout
   f. Return Ok(0)
   ```

8. Update Context command's help text to include guard examples.
