  <behavior>
  **Type changes (assay-types):**
  - Spec gains `criteria: Vec<Criterion>` field, `#[serde(deny_unknown_fields)]`, and description becomes optional (default empty, skip when empty)
  - Criterion gains `#[serde(deny_unknown_fields)]`
  - Schema snapshots must be regenerated after type changes

  **from_str(s: &str) -> Result&lt;Spec, toml::de::Error&gt;**
  - Valid TOML with name + criteria -> Ok(Spec)
  - Valid TOML with name + description + criteria (with and without cmd) -> Ok(Spec)
  - TOML with description omitted -> Ok(Spec { description: "" })
  - Invalid TOML syntax -> Err(toml::de::Error)
  - Unknown key on Spec -> Err mentioning unknown field
  - Unknown key on Criterion -> Err mentioning unknown field

  **validate(spec: &Spec) -> std::result::Result&lt;(), Vec&lt;SpecError&gt;&gt;**
  - Valid spec -> Ok(())
  - Empty name -> Err(vec![SpecError { field: "name", message: "required, must not be empty" }])
  - Whitespace-only name -> same error
  - Zero criteria -> Err(vec![SpecError { field: "criteria", message: "must have at least one criterion" }])
  - Duplicate criterion names -> Err with message identifying the duplicate
  - Empty criterion name -> Err
  - Multiple issues -> all errors collected at once

  **load(path: &Path) -> Result&lt;Spec&gt;**
  - Valid spec file -> Ok(Spec)
  - Missing file -> Err(AssayError::Io)
  - Invalid TOML -> Err(AssayError::SpecParse { path, message })
  - Valid TOML, invalid semantics -> Err(AssayError::SpecValidation { path, errors })

  **scan(specs_dir: &Path) -> Result&lt;ScanResult&gt;**
  - Directory with valid .toml files -> ScanResult { specs: [(slug, Spec), ...], errors: [] }
  - Directory with mix of valid and invalid -> ScanResult { specs: [valid ones], errors: [invalid ones] }
  - Duplicate spec names across files -> errors include duplicate name error
  - Non-.toml files ignored
  - Results sorted by filename (slug)
  - Non-existent directory -> Err(AssayError::SpecScan)

  Cases:
  - `from_str('name = "test"\n[[criteria]]\nname = "c1"\ndescription = "d1"')` -> Ok(Spec { name: "test", description: "", criteria: [Criterion { name: "c1", description: "d1", cmd: None }] })
  - `from_str('name = "test"\ndescription = "desc"\n[[criteria]]\nname = "c1"\ndescription = "d1"\ncmd = "echo hi"')` -> Ok(Spec { ..., criteria: [Criterion { ..., cmd: Some("echo hi") }] })
  - `validate(&Spec { name: "  ", criteria: vec![] })` -> Err(2 errors: empty name + no criteria)
  </behavior>

  **Step 1: Update types in assay-types**

  In `crates/assay-types/src/lib.rs`, update the `Spec` struct:

  ```rust
  #[derive(Debug, Clone, Serialize, Deserialize, JsonSchema)]
  #[serde(deny_unknown_fields)]
  pub struct Spec {
      pub name: String,
      #[serde(default, skip_serializing_if = "String::is_empty")]
      pub description: String,
  ```


  **Step 2: Add error variants to AssayError**

  In `crates/assay-core/src/error.rs`, add three new variants (after existing variants):
  ```rust
  /// Spec file parsing failed (invalid TOML or schema mismatch).
  #[error("parsing spec `{path}`: {message}")]
  SpecParse {
      path: PathBuf,
      message: String,
  },

  /// Spec validation failed (structurally valid TOML but semantically invalid).
  #[error("invalid spec `{path}`:\n{}", .errors.iter().map(|e| format!("  - {e}")).collect::<Vec<_>>().join("\n"))]
  SpecValidation {
      path: PathBuf,
      errors: Vec<crate::spec::SpecError>,
  },

  /// Spec directory scanning failed (I/O error reading the directory).
  #[error("scanning specs directory `{path}`: {source}")]
  SpecScan {
      path: PathBuf,
      source: std::io::Error,
  },
  ```

  **Step 3: Implement spec module**

  Replace stub `crates/assay-core/src/spec/mod.rs` with full implementation. Pattern mirrors `config/mod.rs` exactly:

  - `SpecError` struct (mirrors `ConfigError`): `field: String`, `message: String`, implements `Display`
  - `ScanResult` struct: `specs: Vec<(String, Spec)>` (slug + parsed spec), `errors: Vec<AssayError>` (parse/validation failures per file)
  - `from_str(s: &str) -> std::result::Result<Spec, toml::de::Error>` — one-liner
  - `validate(spec: &Spec) -> std::result::Result<(), Vec<SpecError>>` — checks: name trim non-empty, criteria non-empty, criterion names non-empty, criterion names unique (use HashSet)
  - `load(path: &Path) -> Result<Spec>` — read file, parse, validate, wrap errors with path context
  - `scan(specs_dir: &Path) -> Result<ScanResult>` — read_dir, filter .toml, sort by filename, load each, skip invalid (collect into errors), detect duplicate spec names across files

  For scan: flat directory only (no recursion). Sort entries by filename for deterministic output. Non-.toml files silently skipped. After loading all valid specs, check for duplicate `name` fields across specs and add to errors. Use `std::collections::HashSet` for duplicate detection.
