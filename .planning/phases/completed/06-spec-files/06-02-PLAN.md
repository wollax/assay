Add the `SpecCommand` enum:

```rust
#[derive(Subcommand)]
enum SpecCommand {
    /// Display a spec's criteria in detail
    Show {
        /// Spec name (filename without .toml extension)
        name: String,
        /// Output as JSON instead of table
        #[arg(long)]
        json: bool,
    },
    /// List all available specs
    List,
}
```

**Step 3: Implement the spec show handler**

The handler must:
1. Load config via `assay_core::config::load(&root)` to determine the specs directory
2. Construct the spec file path: `root.join(".assay").join(&config.specs_dir).join(format!("{name}.toml"))`
3. Load the spec via `assay_core::spec::load(&spec_path)`
4. If `--json`: output via `serde_json::to_string_pretty(&spec)` and exit
5. If table format: display a header with spec name and description, then a table of criteria

Table format for `spec show`:
```
Spec: hello-world
Description: A starter spec to verify your Assay setup works

  #  Criterion        Type         Command
  ─  ─────────        ────         ───────
  1  project-builds   executable   echo 'hello from assay'
  2  readme-exists    descriptive
```

Use `println!`-based formatting. No external table library needed.

Color handling:
- "executable" criteria type in green, "descriptive" in yellow/dim
- Respect `NO_COLOR` env var: `fn colors_enabled() -> bool { std::env::var("NO_COLOR").is_err() }`
- Use ANSI escape codes directly (no dependency needed for this simple case): `\x1b[32m` for green, `\x1b[33m` for yellow, `\x1b[0m` for reset

Error handling:
- Config load failure -> print error and exit 1
- Spec not found (Io error with NotFound) -> print `Error: spec '{name}' not found in {specs_dir}` and exit 1
- Spec parse/validation error -> print the error and exit 1

**Step 4: Implement the spec list handler**

The handler must:
1. Load config via `assay_core::config::load(&root)` to determine the specs directory
2. Call `assay_core::spec::scan(&specs_dir_path)`
3. If scan returns specs: print each spec name (one per line), with description if non-empty
4. If scan errors exist: print warnings to stderr for each
5. If no specs found: print "No specs found in {specs_dir}"

List format:
```
Specs:
  hello-world    A starter spec to verify your Assay setup works
  auth-flow      Authentication flow criteria
```

Or if no specs: `No specs found in .assay/specs/`

Error handling:
- Config load failure -> print error and exit 1
- Scan I/O error -> print error and exit 1
