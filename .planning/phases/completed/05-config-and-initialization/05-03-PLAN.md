```rust
/// Options for project initialization.
pub struct InitOptions {
    /// Override the project name (otherwise inferred from directory name).
    pub name: Option<String>,
}

/// Summary of what was created during initialization.
pub struct InitResult {
    pub project_name: String,
    pub created_files: Vec<PathBuf>,
}
```

**Public function:**

```rust
/// Initialize an Assay project at the given root directory.
///
/// Creates `.assay/config.toml`, `.assay/specs/`, `.assay/.gitignore`,
/// and an example spec file.
///
/// Returns `AssayError::AlreadyInitialized` if `.assay/` already exists.
pub fn init(root: &Path, options: &InitOptions) -> Result<InitResult>
```

**Implementation details:**

1. Use `std::fs::create_dir()` (NOT `create_dir_all`) for `.assay/` — returns `AlreadyExists` error which maps to `AssayError::AlreadyInitialized`. Other IO errors map to `AssayError::Io`.
2. Use `std::fs::create_dir()` for `.assay/specs/`.
3. Infer project name via `infer_project_name(root)` helper: `root.file_name().and_then(|n| n.to_str()).filter(|n| !n.trim().is_empty()).unwrap_or("assay-project").to_string()`. If `options.name` is Some, use that instead.
4. Write config.toml using `render_config_template(&project_name)` — string template with comments:
   - `project_name = "{name}"`
   - `specs_dir = "specs/"`
   - Commented-out `[gates]` section with `# default_timeout = 300` and `# working_dir = "."` (with explanatory comments)
5. Write example spec using `render_example_spec()` — static string template for `hello-world.toml`:
   - Two criteria: one with `cmd = "echo 'hello from assay'"`, one without cmd
   - TOML comments explaining each field
6. Write `.gitignore` using `render_gitignore()` — ignores `results/` and `*.cache`
7. Collect all created file paths into `InitResult.created_files`.

**Register module:** Add `pub mod init;` to `crates/assay-core/src/lib.rs`.

**Tests (in the same file, `#[cfg(test)] mod tests`):**

- `test_init_creates_all_artifacts` — init in a tempdir, verify all 4 files/dirs exist
- `test_init_refuses_existing` — init twice, second call returns `AssayError::AlreadyInitialized`
- `test_init_infers_project_name` — init in a dir named "my-cool-project", config.toml contains `project_name = "my-cool-project"`
- `test_init_name_override` — init with `options.name = Some("custom")`, config.toml contains `project_name = "custom"`
- `test_init_config_template_has_comments` — generated config.toml contains `# [gates]` and `# default_timeout`
- `test_init_example_spec_has_both_criteria_modes` — hello-world.toml contains `cmd =` and a criterion without cmd
- `test_infer_project_name_fallback` — when path has no file_name (e.g., `/`), returns "assay-project"

Use `tempfile::tempdir()` or `std::env::temp_dir()` with unique subdirs for filesystem tests. Since `tempfile` is not in workspace deps, use `std::env::temp_dir()` with a UUID-style suffix, or create unique temp dirs manually with `std::fs::create_dir()` and clean up after.

Add an `Init` variant to the `Command` enum in `crates/assay-cli/src/main.rs`:

```rust
/// Initialize a new Assay project in the current directory
Init {
    /// Override the inferred project name
    #[arg(long)]
    name: Option<String>,
},
```

Add a match arm in the `main` function to handle the `Init` command:

```rust
Some(Command::Init { name }) => {
    let root = std::env::current_dir().unwrap_or_else(|e| {
        eprintln!("Error: could not determine current directory: {e}");
        std::process::exit(1);
    });
    let options = assay_core::init::InitOptions { name };
    match assay_core::init::init(&root, &options) {
        Ok(result) => {
            println!("  Created assay project `{}`", result.project_name);
            for path in &result.created_files {
                // Show paths relative to root for clean output
                let display = path.strip_prefix(&root).unwrap_or(path);
                println!("    created {}", display.display());
            }
        }
        Err(e) => {
            eprintln!("Error: {e}");
            std::process::exit(1);
        }
    }
}
```

The output follows cargo-style: `Created assay project \`my-project\`` with one line per created artifact. The error case for AlreadyInitialized produces: `Error: .assay/ already exists. Remove it first to reinitialize.`
