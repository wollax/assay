3. `test_spec_get_params_missing_name` — deserialize `serde_json::json!({})` as `SpecGetParams`. Assert error contains "missing field" and "name".

4. `test_gate_finalize_params_missing_session_id` — deserialize `serde_json::json!({})` as `GateFinalizeParams`. Assert error contains "missing field" and "session_id".

**MCP-02 tests (invalid parameter types):**

5. `test_gate_run_params_invalid_timeout_type` — deserialize `serde_json::json!({"name": "test", "timeout": "abc"})` as `GateRunParams`. Assert error contains "invalid type" or "invalid value".

6. `test_gate_report_params_invalid_passed_type` — deserialize `serde_json::json!({"session_id": "s", "criterion_name": "c", "passed": "yes", "evidence": "e", "reasoning": "r", "evaluator_role": "agent"})` as `GateReportParams`. Assert error contains "invalid type" and "bool".

7. `test_gate_run_params_invalid_include_evidence_type` — deserialize `serde_json::json!({"name": "test", "include_evidence": 42})` as `GateRunParams`. Assert error contains "invalid type" and "bool".

**Key principle:** These tests prove the rmcp serde layer already satisfies MCP-01 and MCP-02 without any custom validation code. The tests serve as regression guards.
