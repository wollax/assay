<task type="auto">
  <name>Task 2: Remove unnecessary clones from gate_run handler (MCP-05)</name>
  <files>crates/assay-mcp/src/server.rs</files>
  <action>
Two significant clones to eliminate in the `gate_run` handler:

**Clone 1: `summary.clone()` at line ~635**

The `summary.clone()` is passed to `history::save_run()` in the else branch (command-only spec, no agent criteria). But `format_gate_response(&summary, include_evidence)` is called earlier at line ~570, after which the original `summary` is only used for:
- `summary.results.clone()` at line 578 (in the if branch)
- `summary.spec_name` at line 639 (in the else branch, for the tracing warn)
- `summary.clone()` at line 635 (the clone we're removing)

**Fix:** In the `else` branch (no agent criteria), extract `summary.spec_name` for the tracing warn before moving `summary` into `save_run`. Restructure to:
```rust
let spec_name_for_log = summary.spec_name.clone();
if let Err(e) = assay_core::history::save_run(
    &assay_dir,
    summary,  // move, not clone
    Some(working_dir.to_string_lossy().to_string()),
    max_history,
) {
    tracing::warn!(
        spec_name = %spec_name_for_log,
        "failed to save command-only gate run history: {e}"
    );
}
```

This trades a `String::clone()` of just the spec name for a `GateRunSummary::clone()` of the entire struct.

**Clone 2: `summary.results.clone()` at line ~578**

This clone passes results to `create_session()`. The `summary.results` is a `Vec<CriterionResult>`. After `format_gate_response()` at line 570, the summary's results are no longer needed by format_gate_response (it already consumed them by reference).

**Fix:** Extract results from summary before passing to create_session. Since `summary` is used later in the if-branch only for `summary.spec_name` (in the session), restructure:
- Call `format_gate_response(&summary, include_evidence)` first (as now)
- Then destructure or extract: `let results = summary.results;` (move) and `let spec_name = summary.spec_name;` (move)
- Use the moved values for `create_session` and the timeout task

However, be careful: `format_gate_response` takes `&summary`, so summary must still be alive for that call. The key insight is that after `format_gate_response` returns, we can move fields out of summary.

**Approach:** After `format_gate_response`, in the `if let Some(info)` branch, move fields out of summary instead of cloning:
```rust
let deterministic_results = summary.results;  // move instead of clone
let spec_name = summary.spec_name;            // move for session + log
```

Then use `spec_name` and `deterministic_results` in `create_session()` and the timeout log.

**Important:** If moving fields out of summary causes issues because other fields are still needed, use partial moves or restructure as needed. The goal is to avoid cloning the `Vec<CriterionResult>` and the entire `GateRunSummary`. Minor String clones (session_id) are acceptable — keep those as-is per research findings.

**Note:** Verify the approach compiles. If partial moves from a struct don't work cleanly in Rust (they do work if you don't use the struct afterward), adjust the approach. The summary struct should not be used after the if/else branch.
