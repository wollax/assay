```rust
/// Change type for files in a clean merge.
#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize, JsonSchema)]
pub enum ChangeType {
    Added,
    Modified,
    Deleted,
}

/// A file changed in a clean merge.
#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize, JsonSchema)]
pub struct FileChange {
    pub path: String,
    pub change_type: ChangeType,
}

/// Classification of a merge conflict.
#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize, JsonSchema)]
pub enum ConflictType {
    Content,
    RenameDelete,
    RenameRename,
    ModifyDelete,
    AddAdd,
    FileDirectory,
    Binary,
    Submodule,
    Other(String),
}

/// A single merge conflict.
#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize, JsonSchema)]
pub struct MergeConflict {
    pub path: String,
    pub conflict_type: ConflictType,
    pub message: String,
}

/// Result of a merge check between two refs.
#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize, JsonSchema)]
pub struct MergeCheck {
    pub clean: bool,
    pub base_sha: String,
    pub head_sha: String,
    #[serde(default, skip_serializing_if = "Option::is_none")]
    pub merge_base_sha: Option<String>,
    pub fast_forward: bool,
    pub ahead: u32,
    pub behind: u32,
    #[serde(default, skip_serializing_if = "Vec::is_empty")]
    pub files: Vec<FileChange>,
    #[serde(default, skip_serializing_if = "Vec::is_empty")]
    pub conflicts: Vec<MergeConflict>,
    pub truncated: bool,
}
```

Create `crates/assay-core/src/merge.rs` with the `merge_check()` function. Follow the `worktree.rs` pattern for git CLI calls.

**Public API:**
```rust
pub fn merge_check(
    project_root: &Path,
    base: &str,
    head: &str,
    max_conflicts: Option<u32>,
) -> Result<MergeCheck>
```

**Internal helper:** Create a local `git_command(args, cwd) -> Result<String>` following the same pattern as `worktree.rs` but using the new `MergeCheckRefError` for ref-related failures, and reusing `WorktreeGit`/`WorktreeGitFailed` for general git failures (these are generic git CLI errors, not worktree-specific despite the name).

**Implementation sequence (from RESEARCH):**
1. **Resolve refs:** `git rev-parse <base>` and `git rev-parse <head>` — resolve both, collect both errors if both fail, produce actionable message
2. **Merge base:** `git merge-base <base_sha> <head_sha>` — may fail for unrelated histories, store as `Option<String>`
3. **Fast-forward:** `git merge-base --is-ancestor <base_sha> <head_sha>` — exit 0 = true, exit 1 = false
4. **Ahead/behind:** `git rev-list --left-right --count <head_sha>...<base_sha>` — parse "N\tM" output into ahead/behind u32 values. Use three-dot notation.
5. **Merge tree:** `git merge-tree --write-tree <base_sha> <head_sha>` — exit 0 = clean, exit 1 = check stdout for valid 40-char hex OID (conflicts) or error. Exit 128 = unrelated histories error.
6. **Clean merge file list:** When clean (exit 0), run `git diff-tree -r --name-status <base_sha> <tree_oid>` to get the list of changed files. Parse status letters: A=Added, M=Modified, D=Deleted.
7. **Conflict parsing:** When conflicted (exit 1 with valid tree OID), parse `CONFLICT (<type>): <message>` lines from stderr/stdout. Map type strings to `ConflictType` enum (handle "content"/"contents" -> `Content`). Extract file path from the message. Apply `max_conflicts` cap (default 20), set `truncated: true` if capped.

**Critical pitfall handling:**
- P1: Exit code 1 disambiguation — check stdout first line for 40-char hex `[0-9a-f]{40}`
- P2: Clean merges need `git diff-tree` follow-up
- P3: "content" and "contents" both map to `ConflictType::Content`
- P4: Exit 128 → return error, not conflicts
- P5: Use `git rev-parse` without `--verify` for relative refs
- P6: `merge-base` failure for unrelated refs → `merge_base_sha: None`

**Unit tests** (in the same file, `#[cfg(test)] mod tests`):
- `test_parse_conflict_type_content` — "content" and "contents" both → `Content`
- `test_parse_conflict_type_variants` — all named variants map correctly
- `test_parse_conflict_type_unknown` — unknown string → `Other(String)`
- `test_parse_change_type` — A/M/D status letters map correctly
- `test_parse_ahead_behind` — "5\t3" → (5, 3)
- `test_is_valid_tree_oid` — 40-char hex = true, shorter/invalid = false

Extract `parse_conflict_type()`, `parse_change_type()`, `parse_ahead_behind()`, and `is_valid_tree_oid()` as private helper functions that the unit tests can test directly.
