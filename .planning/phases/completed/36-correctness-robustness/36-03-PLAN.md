1. Add a constant for the diff budget near the other constants in server.rs:
   ```rust
   /// Maximum byte size for captured git diff (32 KiB).
   const DIFF_BUDGET_BYTES: usize = 32 * 1024;
   ```

2. In the `gate_run` handler, after the eval_future completes and the summary is obtained, but before the `if let Some(info) = agent_info` block, capture the diff. Only capture for specs with agent criteria (inside the `if let Some(info)` block, before creating the session):
   ```rust
   // Capture git diff for the session
   let (diff, diff_truncated, diff_bytes_original) = {
       match std::process::Command::new("git")
           .args(["diff", "HEAD"])
           .current_dir(&working_dir)
           .output()
       {
           Ok(output) if output.status.success() => {
               let raw = String::from_utf8_lossy(&output.stdout);
               if raw.is_empty() {
                   (None, false, None)
               } else {
                   let result = assay_core::gate::truncate_head_tail(&raw, DIFF_BUDGET_BYTES);
                   (
                       Some(result.output),
                       result.truncated,
                       Some(result.original_bytes),
                   )
               }
           }
           Ok(output) => {
               let stderr = String::from_utf8_lossy(&output.stderr);
               tracing::warn!("git diff HEAD failed: {}", stderr.trim());
               (None, false, None)
           }
           Err(e) => {
               tracing::warn!("git diff HEAD command error: {e}");
               (None, false, None)
           }
       }
   };
   ```
   Important: This uses `std::process::Command` directly (not `assay_core::worktree::git_command`) because the error types differ and the MCP crate shouldn't depend on worktree error variants.

   Note: `truncate_head_tail` is `pub(crate)` in `assay-core`, so it needs to be re-exported or the call path adjusted. Since `assay-mcp` depends on `assay-core`, you'll need to either:
   - Make `truncate_head_tail` fully `pub` and re-export from `assay_core::gate`, OR
   - Add a higher-level `pub fn capture_diff(raw: &str, budget: usize) -> (Option<String>, bool, Option<usize>)` in `assay_core::gate` that wraps the truncation logic.

   The cleaner approach: add a public helper in `assay_core::gate::mod.rs`:
   ```rust
   /// Truncate a diff string using head+tail strategy.
   ///
   /// Returns (truncated_output, was_truncated, original_byte_length).
   /// If input is empty, returns (None, false, None).
   pub fn truncate_diff(raw: &str, budget: usize) -> (Option<String>, bool, Option<usize>) {
       if raw.is_empty() {
           return (None, false, None);
       }
       let result = truncate_head_tail(raw, budget);
       (Some(result.output), result.truncated, Some(result.original_bytes))
   }
   ```
   Then call `assay_core::gate::truncate_diff(&raw, DIFF_BUDGET_BYTES)` from the handler.
