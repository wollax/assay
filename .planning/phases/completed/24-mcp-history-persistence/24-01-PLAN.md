<objective>
Fix the integration asymmetry where MCP `gate_run` does not persist history for command-only specs. The CLI always saves via `save_run_record()`, but the MCP handler only saves history through the agent session flow (gate_finalize). Command-only specs (no AgentReport criteria) bypass that flow entirely, so their runs are never recorded.

Add a history save in the `agent_info.is_none()` branch of `gate_run`, plus an integration test that verifies the fix.
</objective>
