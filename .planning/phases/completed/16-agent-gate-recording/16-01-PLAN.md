<objective>
Establish all type foundations needed for agent gate recording. This adds the AgentReport variant to GateKind, creates session/evaluation types (EvaluatorRole, Confidence, AgentSession, AgentEvaluation), extends Criterion and GateCriterion with kind/prompt fields, and adds optional agent evaluation fields to GateResult. All changes use backward-compatible serde attributes. This plan touches only assay-types — no business logic or MCP changes.
</objective>

@crates/assay-types/src/gates_spec.rs (GateCriterion struct — add kind and prompt fields, matching Criterion)
@crates/assay-types/src/gate_run.rs (GateRunRecord — verify no structural changes needed, just confirm compatibility)
@crates/assay-types/src/enforcement.rs (Enforcement enum — pattern reference for derive attributes and serde conventions)

</context>

<name>Task 2: Extend GateKind and GateResult with agent support</name>

<action>

1. Add `AgentReport` variant to `GateKind` enum:
   ```rust
   /// A gate evaluated by an agent via structured reasoning.
   AgentReport,
   ```
   This is a unit variant (no inner data) since it's the static definition — "this criterion is agent-evaluated."

2. Add optional agent fields to `GateResult` struct (after the existing `original_bytes` field):
   ```rust
   /// What the agent observed (concrete facts).
   #[serde(skip_serializing_if = "Option::is_none", default)]
   pub evidence: Option<String>,

   /// Why those facts lead to pass/fail.
   #[serde(skip_serializing_if = "Option::is_none", default)]
   pub reasoning: Option<String>,

   /// Agent's confidence level in the evaluation.
   #[serde(skip_serializing_if = "Option::is_none", default)]
   pub confidence: Option<Confidence>,

   /// Role of the evaluator who produced this result.
   #[serde(skip_serializing_if = "Option::is_none", default)]
   pub evaluator_role: Option<EvaluatorRole>,
   ```
   Import `Confidence` and `EvaluatorRole` from `crate::session`.

4. Add new tests:
   - `gate_kind_agent_report_toml_roundtrip` — `GateKind::AgentReport` serializes with `kind = "AgentReport"` and round-trips
   - `gate_result_agent_fields_json_skipped_when_none` — agent fields omitted from JSON when None
   - `gate_result_agent_fields_json_included_when_set` — agent fields appear in JSON when populated
   - `gate_result_agent_json_roundtrip` — full round-trip with agent fields populated

<name>Task 3: Add kind and prompt fields to Criterion and GateCriterion</name>
<files>crates/assay-types/src/criterion.rs, crates/assay-types/src/gates_spec.rs</files>
<action>
1. In `criterion.rs`, add a `CriterionKind` enum above the `Criterion` struct:
   ```rust
   /// The kind of evaluation a criterion uses.
   #[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize, JsonSchema)]
   pub enum CriterionKind {
       /// Evaluated by an agent via structured reasoning.
       AgentReport,
   }
   ```
   Add an `inventory::submit!` entry with name `"criterion-kind"`.

2. Add two new optional fields to `Criterion` (after `enforcement`):
   ```rust
   /// Criterion evaluation kind. When set to `AgentReport`, this criterion
   /// is evaluated by an agent (not a shell command). Mutually exclusive
   /// with `cmd` and `path`.
   #[serde(skip_serializing_if = "Option::is_none", default)]
   pub kind: Option<CriterionKind>,

   /// Instruction prompt for agent-evaluated criteria.
   /// Provides guidance to the agent on what to evaluate.
   #[serde(skip_serializing_if = "Option::is_none", default)]
   pub prompt: Option<String>,
   ```

3. Add the same two fields to `GateCriterion` in `gates_spec.rs` (after `enforcement`, before `requirements`):

   #[serde(skip_serializing_if = "Option::is_none", default)]
   pub kind: Option<CriterionKind>,

   /// Instruction prompt for agent-evaluated criteria.
   #[serde(skip_serializing_if = "Option::is_none", default)]
   pub prompt: Option<String>,
