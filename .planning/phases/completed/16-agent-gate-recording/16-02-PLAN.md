<action>
1. In `crates/assay-core/src/gate/mod.rs`:
   - Import `CriterionKind` from `assay_types`
   - Update the `evaluate` function to check `criterion.kind` FIRST before `cmd`/`path` and reject AgentReport criteria (they are not evaluable standalone):
     ```rust
     pub fn evaluate(criterion: &Criterion, working_dir: &Path, timeout: Duration) -> Result<GateResult> {
         if criterion.kind == Some(CriterionKind::AgentReport) {
             return Err(AssayError::InvalidCriterion {
                 spec_name: String::new(),
                 criterion_name: criterion.name.clone().unwrap_or_default(),
             }.into());
         }
         match (&criterion.cmd, &criterion.path) {
             (Some(cmd), _) => evaluate_command(cmd, working_dir, timeout),
             (None, Some(path)) => evaluate_file_exists(path, working_dir),
             (None, None) => evaluate_always_pass(),
         }
     }
     ```
   - In `evaluate_all` and `evaluate_all_gates`, when `kind == Some(CriterionKind::AgentReport)`, skip the criterion entirely: set its `result` to `None` with enforcement resolved, and increment `skipped`. These criteria are evaluated later through the session lifecycle (gate_report + gate_finalize), not through the evaluate path.
   - Do NOT add an `evaluate_agent_pending()` function. The standalone `evaluate()` function should return an error if called with `kind == AgentReport` — agent criteria are only meaningful within a session context (managed by the MCP layer), not as standalone evaluations. Add a guard:
     ```rust
     if criterion.kind == Some(CriterionKind::AgentReport) {
         return Err(AssayError::InvalidCriterion {
             spec_name: String::new(),
             criterion_name: criterion.name.clone().unwrap_or_default(),
         }.into());
     }
     ```
   - Similarly update `evaluate_gate_criterion` to check `criterion.kind` before `cmd`/`path` — return an error for AgentReport since it should never reach single-criterion evaluation.
   - Update ALL `GateResult` constructor sites to include the 4 new agent fields as `None` (evidence, reasoning, confidence, evaluator_role). Check: `evaluate_command`, `evaluate_file_exists`, `evaluate_always_pass`, the error-capture path in `evaluate_all`/`evaluate_all_gates`.

2. In `crates/assay-core/src/error.rs`, add new variants:
   ```rust
   /// Session not found (expired, finalized, or never created).
   #[error("session `{session_id}` not found")]
   SessionNotFound { session_id: String },

   /// Criterion name not found in the spec.
   #[error("criterion `{criterion_name}` not found in spec `{spec_name}`")]
   InvalidCriterion { spec_name: String, criterion_name: String },

   /// General session error.
   #[error("session error for `{session_id}`: {message}")]
   SessionError { session_id: String, message: String },
   ```

3. Fix all downstream sites in `assay-cli/src/main.rs` and `assay-mcp/src/server.rs` that construct `Criterion`, `GateCriterion`, or `GateResult` — add the new fields as `None`/default values. Search for struct literal construction of these types in tests and production code.

   - `evaluate_all_with_agent_criterion_marks_as_skipped` — evaluate_all with a mixed spec correctly skips agent criteria (result: None, skipped count incremented)

1. Create `crates/assay-core/src/gate/session.rs` with session lifecycle functions:

   ```rust
   use std::collections::{HashMap, HashSet};
   use chrono::Utc;
   use assay_types::{
       AgentSession, AgentEvaluation, CriterionResult, Enforcement,
       EvaluatorRole, Confidence, GateKind, GateResult, GateRunSummary,
       GateRunRecord, EnforcementSummary,
   };
   use crate::error::{AssayError, Result};
   use crate::history;
   ```

   Functions to implement:

   a. **`create_session`** — Takes spec_name, criteria_names (HashSet<String>), spec_enforcement (HashMap<String, Enforcement>), command_results (Vec<CriterionResult>). Returns an `AgentSession` with a generated session_id (via `history::generate_run_id`), created_at, empty agent_evaluations map.

   b. **`report_evaluation`** — Takes `&mut AgentSession`, criterion_name, AgentEvaluation. Validates:
      - criterion_name is in session.criteria_names (else InvalidCriterion error)
      - Appends the evaluation to session.agent_evaluations[criterion_name]
      Returns Ok(()) on success.

   c. **`finalize_session`** — Takes an `AgentSession`, assay_dir, working_dir, max_history. Builds a `GateRunRecord`:
      - Combines command_results with agent evaluation results
      - For each agent criterion: resolve the effective evaluation using priority (human > independent > self). Take the highest-priority evaluator's result.
      - Build `GateResult` for each agent evaluation with agent fields populated
      - Calculate passed/failed/skipped counts and enforcement summary
      - Agent-reported criteria default to advisory enforcement unless overridden by spec (clamp to spec_enforcement ceiling)
      - Calls `history::save()` to persist
      Returns the GateRunRecord.

   d. **`finalize_as_timed_out`** — Takes an `AgentSession`. Builds a GateRunRecord where any un-evaluated required agent criteria count as failed. Returns the GateRunRecord (does NOT save — caller decides).

   e. **`resolve_evaluator_priority`** helper — Given a Vec<AgentEvaluation>, return the one with highest-priority role (human > independent > self).
