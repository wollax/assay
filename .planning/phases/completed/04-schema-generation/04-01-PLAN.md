**1. Create `crates/assay-types/examples/generate-schemas.rs`:**

The generator binary:
- Resolves the output directory as `schemas/` at the workspace root (use `CARGO_MANIFEST_DIR` env, navigate up two levels to workspace root, join "schemas")
- Creates the `schemas/` directory if it doesn't exist (`fs::create_dir_all`)
- Iterates `schema_registry::all_entries()` — for each entry:
  - Calls `(entry.generate)()` to get the `schemars::Schema`
  - Enriches the schema with `$id` metadata: `schema.insert("$id".to_owned(), format!("https://assay.dev/schemas/{name}.schema.json").into())` where `name` is `entry.name`
  - Serializes with `serde_json::to_string_pretty`
  - Writes to `schemas/{name}.schema.json` with trailing newline: `format!("{json}\n")`
  - Prints progress: `println!("  wrote {}", path.display())`
- Uses `serde_json` (add to example dependencies — or just use it since it's a dev-dependency of the crate; for examples, it needs to be in `[dependencies]` or the example needs its own dep). **Important:** `serde_json` is currently a dev-dependency of assay-types. Examples can use dev-dependencies. Verify this works; if not, move `serde_json` to regular dependencies.

**2. Add `just schemas` and `just schemas-check` recipes to `justfile`:**

```just
# Generate JSON Schema files for all public types
schemas:
    cargo run -p assay-types --example generate-schemas

# Check that generated schemas are up to date (for CI)
schemas-check:
    #!/usr/bin/env bash
    set -euo pipefail
    tmpdir=$(mktemp -d)
    cp -r schemas/ "$tmpdir/schemas-expected"
    just schemas
    if ! diff -r schemas/ "$tmpdir/schemas-expected" > /dev/null 2>&1; then
        echo "ERROR: schemas/ is out of date. Run 'just schemas' and commit."
        rm -rf "$tmpdir"
        exit 1
    fi
    rm -rf "$tmpdir"
    echo "Schemas are up to date."
```

**Note:** Do NOT add `schemas-check` to the `ready` recipe yet — it would create a circular dependency during development (schemas change as types change). It can be added to CI later.
