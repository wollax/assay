<task type="tdd">
<name>Task 2: Crash resilience and full-fidelity roundtrip tests</name>
<files>crates/assay-core/src/history/mod.rs</files>
<action>
Add two tests to `#[cfg(test)] mod tests`:

**Test 1: `test_partial_write_leaves_no_corrupt_file`**

Prove that the atomic write pattern prevents corrupt JSON from appearing in the results directory:

1. Create a `TempDir`
2. Build a valid GateRunRecord
3. Compute the expected final path (same logic as save: `results/<spec>/<run_id>.json`)
4. Write a truncated/invalid JSON string directly to a temp file in the results directory (simulating a crash mid-write — the temp file has a different name than the final path)
5. Verify that `history::list()` does NOT include the temp file (temp files don't have `.json` extension matching run_id format, or are filtered out)
6. Call `history::save()` with the valid record — verify it succeeds
7. Call `history::load()` with the saved run_id — verify it returns valid data

This proves that the tempfile-then-rename pattern means only complete, valid files appear with `.json` extension.

**Test 2: `test_full_fidelity_roundtrip`**

Prove that a GateRunRecord with all fields populated round-trips exactly:

1. Build a GateRunRecord with:
   - Non-trivial run_id and assay_version
   - Specific timestamp (use `DateTime::parse_from_rfc3339` for determinism)
   - `working_dir: Some("/tmp/test-project".into())`
   - `summary` with:
     - Multiple `CriterionResult` entries covering: passed command, failed command, skipped (no result), advisory enforcement, required enforcement
     - Non-zero `passed`, `failed`, `skipped` counts
     - Non-zero `total_duration_ms`
     - Non-default `EnforcementSummary` with all four counters > 0
2. Save to temp dir
3. Load back
4. Assert equality field-by-field (implement or derive PartialEq on GateRunRecord if needed — add `PartialEq` derive to GateRunRecord, GateRunSummary, CriterionResult, and any nested types that don't already have it)
5. Also: read the raw JSON file, deserialize independently with `serde_json::from_str`, confirm the result matches

This proves the phase success criterion: "The persisted GateRunRecord deserializes back to the same logical content that was saved."

**Note on PartialEq:** If GateRunSummary, CriterionResult, or GateResult don't already derive PartialEq, add it. Check each type first — some may already have it. Only add where missing. This is a non-breaking additive change.
</action>
