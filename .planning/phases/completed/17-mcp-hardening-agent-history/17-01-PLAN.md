<name>Task 1: Add timeout parameter and working_dir validation to gate_run</name>
<files>crates/assay-mcp/src/server.rs</files>
<action>
1. Add `timeout` field to `GateRunParams`:
   ```rust
   #[schemars(description = "Maximum seconds to wait for gate evaluation (default: 300). Returns a timeout error if exceeded.")]
   #[serde(default)]
   timeout: Option<u64>,
   ```

2. Add `use std::time::Duration;` to imports.

3. In `gate_run`, after `resolve_working_dir` and before `spawn_blocking`, add path validation:
   ```rust
   if !working_dir.is_dir() {
       return Ok(CallToolResult::error(vec![Content::text(
           format!("working directory does not exist or is not a directory: {}", working_dir.display())
       )]));
   }
   ```

4. Wrap the `spawn_blocking` call with `tokio::time::timeout`:
   ```rust
   let gate_timeout = Duration::from_secs(params.0.timeout.unwrap_or(300));
   let eval_future = tokio::task::spawn_blocking(move || { /* existing evaluate_all logic */ });
   match tokio::time::timeout(gate_timeout, eval_future).await {
       Ok(Ok(summary)) => { /* existing success path */ }
       Ok(Err(e)) => return Err(McpError::internal_error(format!("gate evaluation panicked: {e}"), None)),
       Err(_elapsed) => return Ok(CallToolResult::error(vec![Content::text(
           format!("gate evaluation timed out after {}s", gate_timeout.as_secs())
       )])),
   }
   ```
   Extract `timeout` from params before moving `params.0` fields, since the param struct is consumed.

5. Add a test `test_gate_run_response_with_timeout_field` that verifies GateRunParams deserializes with and without the timeout field.
</action>

<name>Task 2: Surface spec_list scan errors and add enforcement counts to GateRunResponse</name>
<files>crates/assay-mcp/src/server.rs</files>
<action>
1. Add new response structs for spec_list:
   ```rust
   /// Response envelope for spec_list, including scan errors.
   #[derive(Serialize)]
   struct SpecListResponse {
       specs: Vec<SpecListEntry>,
       #[serde(skip_serializing_if = "Vec::is_empty")]
       errors: Vec<SpecListError>,
   }

   /// A spec file that failed to parse during scanning.
   #[derive(Serialize)]
   struct SpecListError {
       message: String,
   }
   ```

2. Update `spec_list` to use the envelope response:
   - After scanning, collect errors from `scan_result.errors` into `Vec<SpecListError>`
   - Serialize `SpecListResponse { specs: entries, errors }` instead of just `entries`

3. Add enforcement count fields to `GateRunResponse`:
   ```rust
   required_passed: usize,
   advisory_passed: usize,
   blocked: bool,
   ```

4. Update `format_gate_response` to compute these fields:
   - `required_passed`: count of results where enforcement is Required and passed is true
   - `advisory_passed`: count of results where enforcement is Advisory and passed is true
   - `blocked`: true when `required_failed > 0`

5. Add/update tests:
   - `test_spec_list_response_serialization` — verify the envelope serializes correctly with and without errors
   - Update `test_format_gate_response_with_evidence` and `test_format_gate_response_summary_only` to assert the new fields (required_passed, advisory_passed, blocked)
</action>
