Create the checkpoint hook script and update hooks.json so checkpoints fire automatically on task operations, compaction, and session stop.
</objective>

<execution_context>
Claude Code hooks are configured in `plugins/claude-code/hooks/hooks.json`. Hook scripts receive JSON on stdin with event-specific fields. The existing `stop-gate-check.sh` and `post-tool-use.sh` scripts demonstrate the patterns.

Key constraints from research:
- PostToolUse hooks have 5s timeout, receive `tool_name`, `tool_input`, `cwd`, `session_id`
- PreCompact hooks are fire-and-forget (command-only, no blocking), receive `hook_event_name`, `trigger`
- Stop hooks CAN block but checkpoint hooks MUST NOT block
- `stop_hook_active` must be checked to prevent infinite loops
- Background execution (`&`) is essential to stay within timeout
- `jq` is required for JSON parsing (guard if missing)
</execution_context>

Create `plugins/claude-code/scripts/checkpoint-hook.sh` with these behaviors:

```bash
#!/usr/bin/env bash
# Checkpoint hook: save team state on task operations, compaction, or stop.
# Fires on PostToolUse[Task|TaskCreate|TaskUpdate], PreCompact, Stop.
# NEVER blocks agent workflow — always exits 0.
```

**Guards (all must pass, exit 0 on any failure):**
1. `jq` must be available (`command -v jq`)
2. Read stdin to `$INPUT`
3. `stop_hook_active` must not be `true` (prevents infinite loops on Stop events)
4. `cwd` must be non-empty and `.assay/` must exist under it
5. `assay` binary must be on PATH (`command -v assay`)

**Debounce:**
1. Check `.assay/checkpoints/.last-checkpoint-ts` for last checkpoint unix timestamp
2. Get current timestamp via `date +%s`
3. If difference < 5 seconds, exit 0 (skip)

**Trigger extraction:**
1. Extract `hook_event_name` from stdin JSON (default: "unknown")
2. Extract `tool_name` from stdin JSON (may be empty for PreCompact/Stop)
3. Build trigger string: `"${EVENT}${TOOL:+:$TOOL}"` (e.g., "PostToolUse:TaskUpdate", "PreCompact", "Stop")
4. Extract `session_id` from stdin JSON (may be empty)

**Execution:**
1. `cd "$CWD"` (exit 0 on failure)
2. Spawn `assay checkpoint save --trigger "$TRIGGER" ${SESSION:+--session "$SESSION"}` in background (`&>/dev/null &`)
3. Exit 0

Make the script executable (`chmod +x`).
