<task type="auto">
  <name>Task 2: Implement budget_context with passthrough and pipeline paths</name>
  <files>crates/assay-core/src/context/budgeting.rs, crates/assay-core/src/context/mod.rs</files>
  <action>
  Create `crates/assay-core/src/context/budgeting.rs` with the following implementation:

  **Public function signature:**
  ```rust
  pub fn budget_context(
      system_prompt: &str,
      spec_body: &str,
      criteria_text: &str,
      diff: &str,
      model_window: u64,
  ) -> Result<Vec<String>, crate::AssayError>
  ```

  **Implementation details:**

  1. **Token estimation:** Use `super::tokens::estimate_tokens_from_bytes` for heuristic token counting. Cast `u64` to `i64` via `as i64` (values will never exceed i64::MAX for realistic content).

  2. **Budget calculation:**
     - `output_reserve = 4_096i64`
     - `max_tokens = model_window as i64`
     - `usable = max_tokens - output_reserve`
     - `safety = (usable as f64 * 0.05) as i64`
     - `target_tokens = usable - safety`

  3. **Passthrough check:** Sum token estimates for all non-empty inputs. If total <= target_tokens, return all content in order `[system_prompt, spec_body (if non-empty), criteria_text, diff (if non-empty)]` without constructing the pipeline. This avoids overhead for the common case.

  4. **Pipeline path** (when content exceeds budget):
     - Build `ContextItem` instances using `ContextItemBuilder`:
       - System prompt: kind=SystemPrompt, pinned=true
       - Criteria: kind=Document, source=Rag, pinned=true
       - Spec body (if non-empty): kind=Document, source=Rag, priority=80
       - Diff (if non-empty): custom kind "Diff", source=Tool, priority=50
     - Build pipeline: `PriorityScorer` + `GreedySlice` + `ChronologicalPlacer`, deduplication disabled, overflow=Truncate
     - Build budget: `ContextBudget::new(max_tokens, target_tokens, output_reserve, HashMap::new(), 5.0)`
     - Execute: `pipeline.execute(items, &budget)`
     - Return: Extract `.content` from each `ScoredItem` in the result, preserving pipeline ordering

  5. **Error mapping:** Map `cupel::CupelError` to `crate::AssayError` using a domain_error or a new variant. Prefer wrapping with `.map_err(|e| crate::AssayError::ContextBudget { message: e.to_string() })` -- add the `ContextBudget` variant to `AssayError` if it doesn't exist, or use an appropriate existing variant.

  **Module wiring:**
  In `crates/assay-core/src/context/mod.rs`:
  - Add `pub mod budgeting;`
  - Add `pub use budgeting::budget_context;`

  **Tests** (in the same file, `#[cfg(test)] mod tests`):

  1. `passthrough_when_content_fits` -- small inputs, verify all content returned in order without truncation
  2. `passthrough_skips_empty_diff` -- empty diff string, verify only 3 items returned (prompt, criteria, spec)
  3. `passthrough_skips_empty_spec_body` -- empty spec body, verify only 3 items returned
  4. `truncates_large_diff` -- create a diff that exceeds budget, verify result contains fewer tokens than input
  5. `pinned_items_always_included` -- large diff + small budget, verify system prompt and criteria are always in result
  6. `empty_everything_returns_empty` -- all empty strings, verify empty vec returned
  7. `budget_calculation_correctness` -- verify the math: for 200k window, output_reserve=4096, safety=5%, target is correct value

  Tests should be deterministic and not depend on cupel internals beyond the public API contract.
