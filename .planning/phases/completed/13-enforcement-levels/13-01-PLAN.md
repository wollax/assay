  <action>
  **1. Create `crates/assay-types/src/enforcement.rs`** with three types:

  **Enforcement enum:**
  ```rust
  #[derive(Debug, Clone, Copy, PartialEq, Default, Serialize, Deserialize, JsonSchema)]
  #[serde(rename_all = "kebab-case")]
  pub enum Enforcement {
      #[default]
      Required,
      Advisory,
  }
  ```
  - Use `Copy` because it's a two-variant fieldless enum read frequently during evaluation.
  - `rename_all = "kebab-case"` matches codebase convention (see `SpecStatus` in `feature_spec.rs`).
  - `Default` = `Required` per CONTEXT.md decision.

  **GateSection struct:**
  ```rust
  #[derive(Debug, Clone, PartialEq, Serialize, Deserialize, JsonSchema)]
  #[serde(deny_unknown_fields)]
  pub struct GateSection {
      #[serde(default)]
      pub enforcement: Enforcement,
  }
  ```
  - `deny_unknown_fields` because this is a user-authored input type.
  - `serde(default)` on enforcement so `[gate]` with no fields defaults to Required.

  **EnforcementSummary struct:**
  ```rust
  #[derive(Debug, Clone, Default, PartialEq, Serialize, Deserialize, JsonSchema)]
  pub struct EnforcementSummary {
      pub required_passed: usize,
      pub required_failed: usize,
      pub advisory_passed: usize,
      pub advisory_failed: usize,
  }
  ```
  - `Default` for zero initialization.
  - No `deny_unknown_fields` — this is an output type.

  Add `inventory::submit!` entries for all three types with names: `"enforcement"`, `"gate-section"`, `"enforcement-summary"`.

  **2. Wire into `crates/assay-types/src/lib.rs`:**
  - Add `pub mod enforcement;` after the existing module declarations.
  - Add `pub use enforcement::{Enforcement, EnforcementSummary, GateSection};` to the re-exports.
  - Add `gate: Option<GateSection>` field to the `Spec` struct with attributes `#[serde(default, skip_serializing_if = "Option::is_none")]`. Place it after `description` and before `criteria`.

  **Important:** The `Spec` struct uses `#[serde(deny_unknown_fields)]`. Adding a new `Option` field with `#[serde(default)]` is safe — existing TOML files without `[gate]` will deserialize with `gate: None`.
  </action>
  <verify>
  Run `cargo check -p assay-types` — must compile. Run `cargo test -p assay-types` — all existing tests must pass (backward compat). Verify the `enforcement.rs` file has all three `inventory::submit!` entries.
  </verify>
