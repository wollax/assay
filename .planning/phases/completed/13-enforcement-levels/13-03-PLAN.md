  <name>Task 1: Update CLI exit code logic and spec new template for enforcement</name>
  <files>
    crates/assay-cli/src/main.rs

  <action>
  **1. Update `StreamCounters` to track enforcement:**

  Add two fields:
  ```rust
  struct StreamCounters {
      passed: usize,
      failed: usize,
      skipped: usize,
      required_failed: usize,
  }
  ```

  Initialize `required_failed: 0` wherever `StreamCounters` is constructed (there are 3 sites: `handle_gate_run`, `handle_gate_run_all`, and in the lambda inside `handle_gate_run_all`).

  **2. Update `stream_criterion()` to accept enforcement context:**

  The function needs to know the resolved enforcement for each criterion to update `required_failed`. Add a parameter `enforcement: Enforcement` (import from `assay_types`).

  In the `Ok(result)` branch where `!result.passed`, check enforcement:
  ```rust
  if !result.passed {
      counters.failed += 1;
      if enforcement == Enforcement::Required {
          counters.required_failed += 1;
      }
  }
  ```

  In the `Err(err)` branch, same:
  ```rust
  counters.failed += 1;
  if enforcement == Enforcement::Required {
      counters.required_failed += 1;
  }
  ```

  **3. Update callers of `stream_criterion()` to pass enforcement:**

  In `handle_gate_run()` and `handle_gate_run_all()`, the criteria loop calls `stream_criterion()`. Before calling it, resolve enforcement:

  For legacy specs (`Spec`):
  ```rust
  let enforcement = assay_core::gate::resolve_enforcement(
      criterion.enforcement,
      spec.gate.as_ref(),  // need access to the Spec
  );
  ```

  `resolve_enforcement` was made `pub fn` in Plan 02 Task 1, so it is already accessible from the CLI crate.

  For the streaming path in `handle_gate_run()`, the criteria are already extracted as `Vec<Criterion>`. The spec/gates is available. Resolve enforcement for each criterion before calling `stream_criterion()`.

  For `handle_gate_run_all()`, the criteria loop works with both Legacy and Directory entries. The gate section reference (`spec.gate` or `gates.gate`) must be available in the loop. Extract it before the criteria loop.

  **Approach for handle_gate_run():**
  ```rust
  let gate_section: Option<&GateSection> = match &entry {
      SpecEntry::Legacy { spec, .. } => spec.gate.as_ref(),
      SpecEntry::Directory { gates, .. } => gates.gate.as_ref(),
  };
  // Then in the loop:
  for criterion in &criteria {
      let enforcement = assay_core::gate::resolve_enforcement(
          criterion.enforcement, gate_section,
      );
      stream_criterion(criterion, &working_dir, cli_timeout, config_timeout, verbose, color, enforcement, &mut counters);
  }
  ```

  Same for `handle_gate_run_all()`.

  **4. Update `print_gate_summary()` exit code:**

  Change:
  ```rust
  if counters.failed > 0 {
      std::process::exit(1);
  }
  ```
  To:
  ```rust
  if counters.required_failed > 0 {
      std::process::exit(1);
  }
  ```

  **5. Update JSON mode exit code in `handle_gate_run()`:**

  Change:
  ```rust
  if summary.failed > 0 {
      std::process::exit(1);
  }
  ```
  To:
  ```rust
  if summary.enforcement.required_failed > 0 {
      std::process::exit(1);
  }
  ```

  **6. Update JSON mode exit code in `handle_gate_run_all()`:**

  Change:
  ```rust
  let any_failed = summaries.iter().any(|s| s.failed > 0);
  ```
  To:
  ```rust
  let any_failed = summaries.iter().any(|s| s.enforcement.required_failed > 0);
  ```

  **7. Update `handle_spec_new()` template:**

  Update the `gates_toml` template to include a `[gate]` section:
  ```rust
  let gates_toml = format!(
      r#"name = "{name}"

  [gate]
  enforcement = "required"

  [[criteria]]
  name = "compiles"
  description = "{name} compiles without errors"
  cmd = "echo 'TODO: add compile check'"
  "#
  );
  ```

  This follows the CONTEXT.md decision: "[gate] section required for new specs."

  **Important:** Do NOT change the pass/fail labels in streaming output. A failing advisory criterion still shows "FAILED" — the distinction is only in whether it triggers exit code 1. The summary line still shows total passed/failed/skipped counts for backward compat.
  </action>
  <verify>
  Run `cargo check -p assay-cli` — must compile. Run `cargo test --workspace` — all tests must pass. Grep for `required_failed` in main.rs to confirm all exit code paths updated.
  </verify>
  <done>
  CLI exit code uses `required_failed > 0` in all three paths (streaming, single-spec JSON, all-specs JSON). `StreamCounters` tracks `required_failed`. `stream_criterion()` accepts and uses enforcement. `spec new` template includes `[gate]` section. All exit code paths updated.
  </done>
