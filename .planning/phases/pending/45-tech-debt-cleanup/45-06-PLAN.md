**1a. Add path-traversal guard to `load_session`** (issue: `load-session-validate-path`):

Before loading a session file, validate that the session ID does not contain path traversal components (`..`, `/`, `\`). Return an error if it does:
```rust
if session_id.contains("..") || session_id.contains('/') || session_id.contains('\\') {
    return Err(/* appropriate error */);
}
```

**1b. Fix `save_session` error context** (issue: `save-session-json-error-context`):

Find the error path in `save_session` that references the directory instead of the file path. Fix the error message to include the actual file path being written.

**1c. Propagate checkpoint timestamp write error** (issue: `checkpoint-timestamp-silent-write`):

In the checkpoint module, find where `.last-checkpoint-ts` is written. If the write error is silently ignored (e.g., `let _ = fs::write(...)`), either:
- Propagate the error if the caller can handle it, OR
- Log a `tracing::warn!` with the error

**1d. Refactor `previous_phase` closure capture** (issue: `previous-phase-capture-fragile`):

Find the fragile closure in `session_update` that captures `previous_phase`. Refactor to be more explicit (e.g., bind the value before the closure, or restructure to avoid the closure).

**1e. Add `truncated` field to `RecoverySummary`** (issue: `recovery-summary-truncated-field`):

Add `pub truncated: bool` to `RecoverySummary` (in assay-core, not assay-types). Set it to `true` when the recovery scan hits the 100-session cap. Add `#[serde(default)]` for backward compat.
