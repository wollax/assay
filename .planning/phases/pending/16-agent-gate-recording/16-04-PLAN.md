<task type="auto">
<name>Task 1: Add gate kind labels to CLI output</name>
<files>crates/assay-cli/src/main.rs</files>
<action>
1. Add a helper function to map GateKind to a display label:
   ```rust
   fn gate_kind_label(kind: &GateKind) -> &'static str {
       match kind {
           GateKind::Command { .. } => "[cmd]",
           GateKind::FileExists { .. } => "[file]",
           GateKind::AlwaysPass => "[auto]",
           GateKind::AgentReport => "[agent]",
       }
   }
   ```

2. Find the gate run output formatting (where criteria results are printed to the terminal). For each criterion result line, prepend the gate kind label. The format should be something like:
   ```
   [cmd]  unit-tests .............. PASS (1.2s)
   [cmd]  lint .................... FAIL (0.8s)
   [file] readme-exists .......... PASS (0.0s)
   [agent] code-review ........... PENDING
   ```
   Use ANSI colors: `[agent]` in cyan/blue to be visually distinct from `[cmd]` in default.

3. For the `history` detail view (the `--detail` or single-record view), when displaying a criterion result that has agent evaluation fields:
   - Show `evaluator_role` if present
   - Show `evidence` (first 200 chars) if present
   - Show `reasoning` (first 200 chars) if present
   - Show `confidence` if present
   Example output:
   ```
   [agent] code-review: PASS (advisory)
     evaluator: self
     confidence: high
     evidence: All functions have doc comments, error types use thiserror...
     reasoning: The code follows the project conventions documented in CLAUDE.md...
   ```

4. Add import for `GateKind` in CLI if not already present.

5. Handle the case where `CriterionResult.result` is `None` but `kind` information isn't directly available on CriterionResult — the label comes from the `GateResult.kind` field when result is `Some`, and for skipped criteria (result is None), infer from the spec if available, or show no label.
</action>
