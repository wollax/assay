# Plan 19-03: Dogfooding Spec

## Goal

Create Assay's own `.assay/` project structure with a `self-check.toml` spec that runs Assay's quality gates on itself, exercising both deterministic (required) and agent (advisory) gate tracks.

## Tasks

<task id="1">
<files>
- .assay/config.toml (new)
- .assay/specs/self-check.toml (new)
- .assay/.gitignore (new)
</files>
<action>
**Create the .assay/ directory structure manually** (do NOT use `assay init` — it creates unwanted example files):

1. Create `.assay/config.toml`:
```toml
project_name = "assay"
```

2. Create `.assay/.gitignore` to exclude transient results but keep config and specs:
```
# Exclude gate run results (transient)
results/
```

3. Create `.assay/specs/self-check.toml` with these criteria:

```toml
name = "self-check"
description = "Assay's own quality gates — dogfooding spec"

[gate]
enforcement = "required"

[[criteria]]
name = "formatting"
description = "Code is formatted with rustfmt"
cmd = "cargo fmt --check"

[[criteria]]
name = "linting"
description = "No clippy warnings"
cmd = "cargo clippy --workspace -- -D warnings"

[[criteria]]
name = "tests"
description = "All tests pass"
cmd = "cargo test --workspace"

[[criteria]]
name = "deny"
description = "Dependency policies pass (licenses, advisories, bans, sources)"
cmd = "cargo deny check"

[[criteria]]
name = "code-quality-review"
description = "Agent reviews code quality and architecture"
kind = "AgentReport"
enforcement = "advisory"
prompt = "Review the codebase for architectural issues, code smells, and potential improvements. Focus on module boundaries, error handling patterns, and test coverage."
```

4. Verify the spec parses correctly:
```bash
cargo run -p assay-cli -- spec get self-check
```

5. Run the self-check gate:
```bash
cargo run -p assay-cli -- gate run self-check
```
This must exit 0 on a clean build (the AgentReport criterion will be skipped since no agent is reporting, which is fine — skipped criteria don't block).

6. Run `just ready` to confirm nothing is broken.

**Gitignore considerations:**
- Check if the repo's root `.gitignore` ignores `.assay/`. If so, add exceptions:
```
!.assay/config.toml
!.assay/specs/
!.assay/.gitignore
```
- The `.assay/results/` directory should remain git-ignored (handled by `.assay/.gitignore`)
</action>
<verify>
- `.assay/config.toml` exists with `project_name = "assay"`
- `.assay/specs/self-check.toml` exists with 5 criteria (4 required + 1 advisory)
- `cargo run -p assay-cli -- spec get self-check` shows the spec content without errors
- `cargo run -p assay-cli -- gate run self-check` exits 0
- `just ready` passes
- `.assay/results/` is git-ignored
</verify>
<done>Dogfooding spec exists. Assay gates itself with fmt, clippy, test, deny (required) and code-quality-review (advisory). `assay gate run self-check` passes on clean build.</done>
</task>
