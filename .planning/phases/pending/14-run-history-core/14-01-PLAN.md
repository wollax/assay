<objective>
Define the GateRunRecord persistence type and build the history module that saves, loads, and lists gate run results as JSON files with atomic writes.

This is the infrastructure layer for HIST-01 (persist results) and HIST-04 (atomic writes). After this plan, callers of evaluate_all() can construct a GateRunRecord from the returned GateRunSummary and persist it via history::save(). Phase 15 CLI and Phase 17 MCP will consume history::load() and history::list().
</objective>

```rust
/// A complete, versioned record of a single gate evaluation run.
///
/// Wraps `GateRunSummary` with metadata for persistence and audit.
/// Uses `deny_unknown_fields` — records are versioned artifacts;
/// field mismatches should fail loudly. `assay_version` supports
/// future schema migration.
#[derive(Debug, Clone, Serialize, Deserialize, JsonSchema)]
#[serde(deny_unknown_fields)]
pub struct GateRunRecord {
    /// Unique run identifier: `<timestamp>-<6-char-hex>` (e.g., `20260304T223015Z-a3f1b2`).
    pub run_id: String,
    /// Version of assay that produced this record (from `env!("CARGO_PKG_VERSION")`).
    pub assay_version: String,
    /// UTC timestamp when the evaluation started.
    pub timestamp: chrono::DateTime<chrono::Utc>,
    /// Working directory used for evaluation, if available.
    #[serde(default, skip_serializing_if = "Option::is_none")]
    pub working_dir: Option<String>,
    /// The complete gate run summary with all criterion results.
    pub summary: GateRunSummary,
}
```

Add `chrono` import at top of file (it's already a workspace dep of assay-types via schemars chrono04 feature — check if direct dep is needed, if so add `chrono.workspace = true` to `crates/assay-types/Cargo.toml`).

Add re-export in `crates/assay-types/src/lib.rs`:
- Add `GateRunRecord` to the `pub use gate_run::{...}` line.

Register schema via `inventory::submit!` for `gate-run-record`, same pattern as existing types.

**Step 2: Create history module**

Create `crates/assay-core/src/history/mod.rs` with three public functions and one helper:

1. `generate_run_id(timestamp: DateTime<Utc>) -> String`
   - Format timestamp as `%Y%m%dT%H%M%SZ` using `chrono::DateTime::format()`
   - Generate 6-char lowercase hex suffix using `std::collections::hash_map::RandomState`:
     ```rust
     use std::collections::hash_map::RandomState;
     use std::hash::{BuildHasher, Hasher};
     let random = RandomState::new().build_hasher().finish();
     let suffix = format!("{:06x}", random & 0xFF_FFFF);
     ```
   - Return `format!("{timestamp_str}-{suffix}")`

2. `pub fn save(assay_dir: &Path, record: &GateRunRecord) -> Result<PathBuf>`
   - Compute results dir: `assay_dir.join("results").join(&record.summary.spec_name)`
   - `std::fs::create_dir_all(&results_dir)?`
   - Serialize: `serde_json::to_string_pretty(record)?`
   - Create temp file in same directory: `NamedTempFile::new_in(&results_dir)?`
   - Write JSON bytes to temp file
   - `tmpfile.as_file().sync_all()?` (flush to disk before rename)
   - Compute final path: `results_dir.join(format!("{}.json", record.run_id))`
   - `tmpfile.persist(&final_path)?` (atomic rename)
   - Return `Ok(final_path)`

3. `pub fn load(assay_dir: &Path, spec_name: &str, run_id: &str) -> Result<GateRunRecord>`
   - Build path: `assay_dir.join("results").join(spec_name).join(format!("{run_id}.json"))`
   - Read file contents with `std::fs::read_to_string`
   - Deserialize with `serde_json::from_str` (deny_unknown_fields will enforce strictness)
   - Return the record or map IO/serde errors to AssayError

4. `pub fn list(assay_dir: &Path, spec_name: &str) -> Result<Vec<String>>`
   - Build path: `assay_dir.join("results").join(spec_name)`
   - If directory doesn't exist, return `Ok(vec![])` (not an error)
   - Read directory entries, filter for `.json` extension, extract stem as run_id
   - Sort alphabetically (timestamp prefix makes this chronological)
   - Return sorted vec

Add appropriate error variants to `crates/assay-core/src/error.rs` if needed (check if existing variants cover IO and serde_json errors; add `History` or `Persist` variant if not — wrap `serde_json::Error` and `tempfile::PersistError`).

**Step 4: Add unit tests in history/mod.rs**

Include `#[cfg(test)] mod tests` with:

1. `test_generate_run_id_format` — verify format matches `YYYYMMDDTHHMMSSZ-[0-9a-f]{6}` regex pattern
2. `test_save_creates_file` — save a record, verify file exists at expected path
3. `test_save_creates_directories` — save to non-existent spec dir, verify it's auto-created
4. `test_load_roundtrip` — save then load, verify all fields match
5. `test_list_empty_dir` — list for non-existent spec returns empty vec
6. `test_list_returns_sorted` — save multiple records, verify list returns them sorted
7. `test_save_does_not_clobber` — save two records with same spec, both files exist

All tests should use `tempfile::TempDir` for isolation.

For constructing test GateRunSummary values, build them manually with known field values (spec_name, passed/failed/skipped counts, empty results vec, zero duration, default enforcement summary).
