**Extractor** (`extractor.rs`):

Public function `extract_team_state(project_dir: &Path, session_id: Option<&str>, trigger: &str) -> crate::Result<TeamCheckpoint>`:
1. Use `context::find_session_dir(project_dir)` to locate session dir
2. Use `context::resolve_session(&session_dir, session_id)` to get the session JSONL path
3. Use `context::parse_session(&path)` to get parsed entries
4. Extract session_id from the file stem
5. Scan entries to build agent list:
   - Primary agent: always present, model from last assistant entry's `message.model`
   - Subagents: entries where `meta.is_sidechain == true`, unique by agentId (extracted from ProgressEntry data field: `data.get("agentId")`)
6. Scan progress entries for TaskCreate/TaskUpdate tool uses (see research code example):
   - Navigate `data.pointer("/message/message/content")` for content array
   - Match `ContentBlock`-like objects where `name` is "TaskCreate" or "TaskUpdate"
   - Build `TaskState` entries from input fields (subject, description, taskId, status)
   - Track task-to-agent assignment via `agentId` in the progress entry data
7. Scan system entries for compact boundaries:
   - Check `data` for `subtype: "compact_boundary"` or `type: "compact_boundary"`
   - Extract `compactMetadata.trigger` and `compactMetadata.preTokens`
8. Build context health from `context::quick_token_estimate(&path)`:
   - If usage data available, populate `ContextHealthSnapshot` with token counts and utilization
9. Assemble and return `TeamCheckpoint`

Internal helper `extract_agents(entries: &[ParsedEntry]) -> Vec<AgentState>`:
- Tracks seen agent IDs, builds primary + sidechain agents
- Sets status based on recency of last entry (heuristic: active if last entry within session)

Internal helper `extract_tasks(entries: &[ParsedEntry]) -> Vec<TaskState>`:
- Builds task list from TaskCreate, updates status from TaskUpdate
- Tasks are identified by sequential index (TaskCreate order) or by taskId in TaskUpdate

Internal helper `extract_compaction(entries: &[ParsedEntry]) -> (Option<String>, Option<String>)`:
- Returns (last_compaction_timestamp, compaction_trigger)

**Team config discovery** (`config.rs`):

Public function `discover_team_config() -> Option<TeamConfigContext>`:
1. Resolve `~/.claude/teams/` via `dirs::home_dir()`
2. If `~/.claude/teams/*/config.json` exists for any team dir, read and deserialize as `serde_json::Value`
3. If `~/.claude/teams/*/inboxes/*.json` exist, collect as coordination context entries
4. Return `None` gracefully when neither config.json nor inboxes exist
5. Return `Some(TeamConfigContext { config, inbox_entries })` when data is found

Struct `TeamConfigContext` (internal to config.rs):
- `config: Option<serde_json::Value>` — parsed config.json if present
- `inbox_entries: Vec<serde_json::Value>` — inbox messages for coordination context

Public function `merge_team_config(checkpoint: &mut TeamCheckpoint, team_config: Option<&TeamConfigContext>)`:
- If `team_config` is `None`, no-op (pure session-extracted checkpoint)
- If `team_config.config` is `Some`, use config as base and let session-extracted data enrich/override (config priority for static fields, session data priority for runtime state like agent status and task progress)
- Inbox entries are informational — store count or last-message timestamp if useful, but do not alter core checkpoint fields

The extractor's `extract_team_state()` calls `discover_team_config()` and `merge_team_config()` as a final enrichment step before returning the checkpoint.

**Persistence** (`persistence.rs`):

Public function `save_checkpoint(assay_dir: &Path, checkpoint: &TeamCheckpoint) -> crate::Result<PathBuf>`:
1. Create `.assay/checkpoints/` and `.assay/checkpoints/archive/` dirs if needed
2. Render checkpoint as JSON frontmatter (between `---` delimiters) + markdown body (use `render_checkpoint` helper). Note: frontmatter is serialized as JSON, not YAML — JSON is a valid YAML subset so tooling that reads YAML frontmatter will still work.
3. Write atomically to `latest.md` (tempfile in same dir, then rename)
4. Write atomically to `archive/{timestamp}.md` (ISO 8601 filename, colons replaced with dashes for filesystem compat)
5. Prune archive: list files, sort by name, delete oldest beyond 50
6. Update `.last-checkpoint-ts` file with current unix timestamp
7. Return the archive path

Public function `load_latest_checkpoint(assay_dir: &Path) -> crate::Result<TeamCheckpoint>`:
1. Read `checkpoints/latest.md`
2. Split on `---` delimiters to extract frontmatter
3. Parse frontmatter as JSON into `TeamCheckpoint`
4. Return the checkpoint

Public function `list_checkpoints(assay_dir: &Path, limit: usize) -> crate::Result<Vec<CheckpointEntry>>`:
1. Read `checkpoints/archive/` directory
2. List `.md` files, sorted by name descending (newest first)
3. For each file up to limit, extract frontmatter and return `CheckpointEntry { path, timestamp, trigger, agent_count, task_count }`

Internal helper `render_checkpoint(checkpoint: &TeamCheckpoint) -> String`:
- Serialize checkpoint as JSON for frontmatter (between `---` delimiters)
- Render human-readable markdown body: heading, metadata, agents table, tasks table, context health section
- See research doc for exact format

Create a simple `CheckpointEntry` struct (in persistence.rs, not in assay-types — it's a persistence-layer concern):
- `path: PathBuf`
- `timestamp: String`
- `trigger: String`
- `agent_count: usize`
- `task_count: usize`
