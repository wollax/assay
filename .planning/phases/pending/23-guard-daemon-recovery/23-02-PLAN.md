```rust
//! Circuit breaker preventing infinite recovery loops.
//!
//! Tracks recovery timestamps in a sliding window. Trips when
//! max_recoveries is exceeded within recovery_window_secs.

use std::collections::VecDeque;
use std::time::{Duration, Instant};

use assay_types::context::PrescriptionTier;

/// Circuit breaker state machine for guard daemon recovery limiting.
pub struct CircuitBreaker {
    /// Maximum recoveries allowed before tripping.
    max_recoveries: u32,
    /// Time window for counting recoveries.
    window: Duration,
    /// Timestamps of recent recovery attempts (oldest first).
    recoveries: VecDeque<Instant>,
    /// Whether the breaker has tripped.
    tripped: bool,
}
```

2. Implement methods:
   - `new(max_recoveries: u32, window_secs: u64) -> Self`: Initialize with empty recoveries, tripped = false.
   - `record_recovery(&mut self) -> u32`: Push `Instant::now()`, prune entries older than window, return current count of recoveries in window.
   - `should_trip(&self) -> bool`: Return true if recoveries in window >= max_recoveries.
   - `is_tripped(&self) -> bool`: Return self.tripped.
   - `trip(&mut self)`: Set tripped = true.
   - `recovery_count(&self) -> u32`: Count of recoveries currently in window.
   - `current_tier(&self) -> PrescriptionTier`: Map recovery count to escalating tier:
     - 0 or 1 recovery -> Gentle
     - 2 recoveries -> Standard
     - 3+ recoveries -> Aggressive
   - `reset_if_quiet(&mut self)`: Prune old entries. If no recoveries remain in window, reset tripped to false. This allows the daemon to resume after a quiet period.
   - `prune_old(&mut self)`: Internal helper. Remove entries from front of VecDeque where `entry.elapsed() > self.window`.

3. Important: The `record_recovery` method should call `prune_old` first, then push the new timestamp, then check `should_trip`. If tripped, call `self.trip()`.

4. If `crates/assay-core/src/guard/mod.rs` does not exist yet (plan 01 may not have run), create it with:
   ```rust
   //! Guard daemon: background context protection with threshold-based pruning.
   pub mod circuit_breaker;
   ```
   If it already exists from plan 01, just add `pub mod circuit_breaker;` to it.

5. Similarly, if `crates/assay-core/src/lib.rs` doesn't have `pub mod guard;`, add it. (Plan 01 should have added it, but be defensive.)

6. Write comprehensive tests:
   - `new_breaker_not_tripped`: Fresh breaker has recovery_count 0, not tripped.
   - `single_recovery_not_tripped`: One recovery doesn't trip (default max=3).
   - `max_recoveries_trips`: 3 recoveries in window -> should_trip returns true.
   - `old_recoveries_pruned`: Record recovery, simulate time passage (use a helper that inserts old Instant), verify old ones are pruned.
   - `escalation_gentle`: 1 recovery -> Gentle tier.
   - `escalation_standard`: 2 recoveries -> Standard tier.
   - `escalation_aggressive`: 3 recoveries -> Aggressive tier.
   - `reset_after_quiet`: Trip the breaker, then call reset_if_quiet after all entries expire -> tripped becomes false, recovery_count is 0.
   - `recovery_count_accurate`: Add 3 recoveries, verify count is 3. Prune old, verify updated count.

   For time-sensitive tests, add a `#[cfg(test)]` helper method `record_recovery_at(&mut self, instant: Instant)` that inserts a specific Instant instead of `Instant::now()`. This avoids flaky tests that depend on wall-clock timing.
