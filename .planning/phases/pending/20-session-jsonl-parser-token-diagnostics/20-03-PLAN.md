<task type="auto">
  <name>Task 1: Add Context subcommand to CLI</name>
  <files>crates/assay-cli/src/main.rs</files>
  <action>
  Add `Context` variant to the `Command` enum following the existing nested subcommand pattern (like `Spec` and `Gate`):

  ```rust
  /// Context window diagnostics for Claude Code sessions
  #[command(after_long_help = "\
  Examples:
    Diagnose the most recent session:
      assay context diagnose

    Diagnose a specific session:
      assay context diagnose 3201041c-df85-4c91-a485-7b8c189f7636

    List recent sessions:
      assay context list

    List with token counts:
      assay context list --tokens")]
  Context {
      #[command(subcommand)]
      command: ContextCommand,
  },
  ```

  Define `ContextCommand` enum:
  ```rust
  #[derive(Subcommand)]
  enum ContextCommand {
      /// Analyze token usage and bloat in a Claude Code session
      Diagnose {
          /// Session ID (defaults to most recent session)
          session_id: Option<String>,
          /// Output as JSON
          #[arg(long)]
          json: bool,
          /// Plain output (no color, no Unicode symbols)
          #[arg(long)]
          plain: bool,
      },
      /// List Claude Code sessions with metadata
      List {
          /// Maximum number of sessions to show
          #[arg(long, default_value = "20")]
          limit: usize,
          /// Show all sessions (overrides --limit)
          #[arg(long)]
          all: bool,
          /// Include token counts (slower, reads session tails)
          #[arg(long)]
          tokens: bool,
          /// Output as JSON
          #[arg(long)]
          json: bool,
          /// Plain output (no color, no Unicode symbols)
          #[arg(long)]
          plain: bool,
      },
  }
  ```

  Wire up the command dispatch in the main match:
  ```rust
  Some(Command::Context { command }) => match command {
      ContextCommand::Diagnose { session_id, json, plain } => {
          cmd_context_diagnose(session_id, json, plain)?;
      }
      ContextCommand::List { limit, all, tokens, json, plain } => {
          cmd_context_list(if all { usize::MAX } else { limit }, tokens, json, plain)?;
      }
  },
  ```
  </action>
  <verify>

  </verify>
  <done>Context subcommand compiles with Diagnose and List sub-subcommands</done>

<task type="auto">
  <name>Task 2: Implement diagnose and list display functions</name>
  <files>crates/assay-cli/src/main.rs</files>
  <action>
  Implement `cmd_context_diagnose()`:

  1. Resolve the current project directory (use `std::env::current_dir()`)
  2. Find the session directory via `assay_core::context::find_session_dir()`
  3. Resolve the session file via `assay_core::context::resolve_session()`
  4. Call `assay_core::context::diagnose()` to get the `DiagnosticsReport`
  5. If `--json`, serialize and print the report as JSON
  6. Otherwise, format as a dashboard:

  **Dashboard layout (with color, unless --plain):**

  ```
  ── Context Diagnostics ─────────────────────────────
  Session:  3201041c-df85-4c91-a485-7b8c189f7636
  Model:    claude-opus-4-6
  File:     ~/.claude/projects/-Users-.../session.jsonl (2.4 MB)

  ── Overview ─────────────────────────────────────────
  Context tokens:    156,234 / 200,000 (78.1%)   ⚠ Warning
  Output tokens:     12,450
  Total entries:     830 (72 messages)
  Skipped entries:   0

  ── Bloat Breakdown ──────────────────────────────────
  Category            Bytes      Count    % of File
  Progress ticks      1,234,567  640      48.2%
  Tool output         567,890    42       22.1%
  Thinking blocks     234,567    9        9.1%
  Stale reads         123,456    8        4.8%
  System reminders    45,678     12       1.8%
  Metadata            23,456     15       0.9%
  ```

  Use the existing ANSI color pattern from the codebase. Green for healthy (<60%), yellow for warning (60-85%), red for critical (>85%).

  For `--plain`, strip all ANSI codes and replace Unicode symbols with ASCII equivalents (e.g., `--` instead of `──`, `[OK]` instead of checkmark).

  Implement `cmd_context_list()`:

  1. Call `assay_core::context::list_sessions(None, limit, include_tokens)`
  2. If `--json`, serialize and print as JSON array
  3. Otherwise, format as a table:

  ```
  Session ID                            Size      Entries  Modified
  3201041c-df85-4c91-a485-7b8c189f7636  2.4 MB    830      2h ago
  a1b2c3d4-e5f6-7890-abcd-ef1234567890  1.1 MB    412      1d ago
  ...
  ```

  If `--tokens` is set, add a `Tokens` column:
  ```
  Session ID                            Size      Entries  Tokens    Modified
  3201041c-df85-4c91-a485-7b8c189f7636  2.4 MB    830      156,234   2h ago
  ```

  Use human-friendly size formatting (KB, MB) and relative time for modification dates.
