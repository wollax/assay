**Create `crates/assay-types/src/gate.rs`:**

Define `GateKind` enum:
- `#[derive(Debug, Clone, PartialEq, Serialize, Deserialize, JsonSchema)]`
- `#[serde(tag = "kind")]` — internal tagging
- Variants: `Command { cmd: String }` and `AlwaysPass` (unit variant)
- Doc comments on the enum and each variant (see research code examples)

Define `GateResult` struct:
- `#[derive(Debug, Clone, Serialize, Deserialize, JsonSchema)]`
- Do NOT derive `PartialEq` (DateTime equality is semantically questionable)
- Fields:
  - `passed: bool`
  - `kind: GateKind` — self-describing, per CONTEXT.md decision
  - `stdout: String` with `#[serde(skip_serializing_if = "String::is_empty")]` and `#[serde(default)]`
  - `stderr: String` with same serde attrs
  - `exit_code: Option<i32>` with `#[serde(skip_serializing_if = "Option::is_none")]` and `#[serde(default)]`
  - `duration_ms: u64`
  - `timestamp: DateTime<Utc>` (from `chrono::{DateTime, Utc}`)
- Doc comments on the struct and each field

Add `#[cfg(test)] mod tests` in gate.rs with these tests:
1. `gate_kind_command_toml_roundtrip` — serialize Command variant to TOML string, assert contains `kind = "Command"` and `cmd = "cargo test"`, deserialize back, re-serialize, assert strings match
2. `gate_kind_always_pass_toml_roundtrip` — same for AlwaysPass variant, assert contains `kind = "AlwaysPass"`
3. `gate_result_json_skips_empty_fields` — create GateResult with empty stdout/stderr and None exit_code, serialize to JSON, assert stdout/stderr/exit_code keys are absent
4. `gate_result_json_includes_populated_fields` — create GateResult with non-empty stdout/stderr and Some(0) exit_code, serialize to JSON, assert those keys ARE present

**Create `crates/assay-types/src/criterion.rs`:**

Define `Criterion` struct:
- `#[derive(Debug, Clone, PartialEq, Serialize, Deserialize, JsonSchema)]`
- Fields:
  - `name: String`
  - `description: String`
  - `cmd: Option<String>` with `#[serde(skip_serializing_if = "Option::is_none")]` and `#[serde(default)]`
- Doc comments on struct and fields (mention forward-compatibility with future `prompt` field in comment)

Add `#[cfg(test)] mod tests` in criterion.rs:

2. `criterion_cmd_some_is_valid` — create Criterion with cmd, serialize to TOML, assert cmd present, roundtrip
