<feature>
  <name>Config loading and validation</name>
  <files>crates/assay-core/src/config/mod.rs</files>
  <behavior>
  **from_str(s: &str) -> Result&lt;Config, toml::de::Error&gt;**
  - Valid TOML with all fields -> Ok(Config { project_name, specs_dir, gates })
  - Valid TOML with only project_name -> Ok(Config { project_name, specs_dir: "specs/", gates: None })
  - Invalid TOML syntax -> Err(toml::de::Error with line/column)
  - Unknown key in TOML -> Err(toml::de::Error mentioning unknown field)

  **validate(config: &Config) -> std::result::Result&lt;(), Vec&lt;ConfigError&gt;&gt;**
  - Valid config -> Ok(())
  - Empty project_name -> Err(vec![ConfigError { field: "project_name", message: "required, must not be empty" }])
  - Whitespace-only project_name -> same error
  - Empty specs_dir -> Err(vec![ConfigError { field: "specs_dir", ... }])
  - default_timeout = 0 -> Err(vec![ConfigError { field: "[gates].default_timeout", ... }])
  - Multiple invalid fields -> Err(vec![...all errors...]) (not just the first)

  **load(root: &Path) -> Result&lt;Config&gt;**
  - Valid config.toml at root/.assay/config.toml -> Ok(Config)
  - Missing file -> Err(AssayError::Io)
  - Invalid TOML -> Err(AssayError::ConfigParse { path, message })
  - Valid TOML, invalid semantics -> Err(AssayError::ConfigValidation { path, errors })

  Cases:
  - `from_str('project_name = "test"')` -> Ok(Config { project_name: "test", specs_dir: "specs/", gates: None })
  - `from_str('project_name = "test"\nspecs_dir = "my-specs/"')` -> Ok(Config { ..., specs_dir: "my-specs/" })
  - `from_str('project_name = "test"\n[gates]\ndefault_timeout = 600')` -> Ok(Config { ..., gates: Some(GatesConfig { default_timeout: 600, working_dir: None }) })
  - `from_str('unknown_key = "x"')` -> Err (unknown field)
  - `validate(&Config { project_name: "".into(), specs_dir: "".into(), gates: Some(GatesConfig { default_timeout: 0, working_dir: None }) })` -> Err(3 errors)
  </behavior>
  <implementation>
  Implement in `crates/assay-core/src/config/mod.rs`. The ConfigError type should already exist from Plan 01.

  **from_str:** One-liner wrapping `toml::from_str::<Config>(s)`. Returns raw `toml::de::Error` (not wrapped in AssayError). This is the composable API: tests and tools can skip validation.

  **validate:** Collect errors into a `Vec<ConfigError>`. Check: project_name.trim().is_empty(), specs_dir.trim().is_empty(), gates.default_timeout == 0. Return Ok(()) if no errors, Err(errors) if any.

  **load:** Read file at `root.join(".assay/config.toml")`, parse with `toml::from_str`, wrap parse errors in `AssayError::ConfigParse`, call `validate()`, wrap validation errors in `AssayError::ConfigValidation`. Return Ok(config) on success.

  Re-export `config` module's public API from the config module. Ensure `ConfigError` is exported so `AssayError::ConfigValidation` consumers can access it.
  </implementation>
</feature>
