**CORE-09: Scan warning emission**

5. In `scan()` (line 502), after building the `ScanResult`, emit warnings for any errors before returning:
```rust
for err in &errors {
    eprintln!("Warning: {err}");
}
```
Add this just before the `Ok(ScanResult { ... })` return. This ensures callers that forget to iterate `.errors` still get warnings. Existing callers that iterate `.errors` will show duplicates, so either:
- Remove the caller-side warning loops (preferred — centralizes warning in core), OR
- Add a `warnings_emitted: bool` field or similar

**Recommended approach:** Add the warning emission in `scan()` and leave the `.errors` field for callers that need programmatic access (MCP uses it for structured responses). The CLI callers can drop their `for err in &result.errors` loops since core now handles it. However, since this plan only modifies assay-core files, just add the `eprintln!` in `scan()` — CLI cleanup is out of scope.

```rust
pub fn evaluate_all(...) -> GateRunSummary {
    let criteria: Vec<(Criterion, Enforcement)> = spec.criteria.iter().map(|c| {
        let enforcement = resolve_enforcement(c.enforcement, spec.gate.as_ref());
        (c.clone(), enforcement)
    }).collect();
    evaluate_criteria(&spec.name, criteria, working_dir, cli_timeout, config_timeout)
}
```

3. Refactor `evaluate_all_gates()` similarly:
```rust
pub fn evaluate_all_gates(...) -> GateRunSummary {

        (to_criterion(gc), enforcement)
    }).collect();
    evaluate_criteria(&gates.name, criteria, working_dir, cli_timeout, config_timeout)
}
```
