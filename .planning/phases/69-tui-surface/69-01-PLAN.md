  <files>crates/assay-tui/src/gate_wizard.rs, crates/assay-tui/src/lib.rs</files>
  <action>
Create `crates/assay-tui/src/gate_wizard.rs` following the exact patterns from `wizard.rs`.

**Public types:**

1. `GateWizardState` struct (all fields `pub(crate)` except `pub error: Option<String>`):
   - `step: usize` — current wizard step (0-6)
   - `fields: Vec<Vec<String>>` — per-step text input buffers (steps 0, 1 use these)
   - `cursor: usize` — cursor position in active text input
   - `available_gates: Vec<String>` — gate slugs loaded at wizard open time for extends single-select
   - `extends_list_state: ListState` — ratatui list state for extends step
   - `available_libs: Vec<CriteriaLibrary>` — loaded at wizard open for includes multi-select
   - `includes_list_state: ListState` — ratatui list state for includes step
   - `selected_includes: HashSet<usize>` — indices into available_libs that are toggled on
   - `criteria: Vec<CriterionInput>` — collected criteria so far
   - `criteria_sub_step: CriteriaSubStep` — which field within the criteria loop is active
   - `preconditions_active: bool` — whether user said "yes" to preconditions prompt
   - `precondition_requires: Vec<String>` — collected requires slugs
   - `precondition_commands: Vec<String>` — collected command strings
   - `precondition_sub_step: PreconditionSubStep` — Ask | Requires | Commands
   - `edit_slug: Option<String>` — None = create mode, Some(slug) = edit mode
   - `error: Option<String>` — inline error display
   - `auto_skip_msg: Option<String>` — transient message when a list step is auto-skipped (empty list)

2. `CriteriaSubStep` enum: `Name`, `Description`, `Cmd`, `AddAnother`

3. `PreconditionSubStep` enum: `Ask`, `Requires`, `Commands`

4. `GateWizardAction` enum: `Continue`, `Submit(GateWizardInput)`, `Cancel`

**Constructors:**

- `GateWizardState::new(available_gates: Vec<String>, available_libs: Vec<CriteriaLibrary>) -> Self` — creates fresh state at step 0 with `fields: vec![vec![String::new()]]`, extends_list_state selecting index 0 (the "(none)" option), empty selected_includes, edit_slug = None.

- `GateWizardState::from_existing(gates: &GatesSpec, slug: String, available_gates: Vec<String>, available_libs: Vec<CriteriaLibrary>) -> Self` — edit mode constructor:
  - Pre-fills `fields[0]` with gate name, `fields[1]` with description (or empty)
  - Sets `extends_list_state` selection to match `gates.extends` in available_gates (offset by 1 for "(none)" at index 0), or 0 if None
  - Populates `selected_includes` by matching `gates.include` slugs against `available_libs[i].name`
  - Converts existing criteria from `gates.criteria` into `Vec<CriterionInput>` (map name, description, cmd)
  - If `gates.preconditions` is Some, populates precondition_requires and precondition_commands from it
  - Sets `edit_slug = Some(slug)`

**Step flow (7 steps):**
- Step 0: Name (text input, required — blank Enter shows error "Gate name is required")
- Step 1: Description (text input, optional — blank Enter advances)
- Step 2: Extends (ListState single-select — "(none)" as first item, then available_gates; auto-skips with message if available_gates is empty; Up/Down navigate, Enter selects)
- Step 3: Includes (toggle list — Space toggles, Up/Down navigate, Enter confirms; auto-skips if available_libs is empty)
- Step 4: Criteria loop (CriteriaSubStep cycle: Name → Description → Cmd → AddAnother; blank name in AddAnother ends loop; in edit mode, walk existing criteria first with `d` to delete, then AddAnother for new ones)
- Step 5: Preconditions (PreconditionSubStep: Ask "Add preconditions? (y/N)" → if yes: Requires loop (blank Enter ends) → Commands loop (blank Enter ends); if no: advance to step 6)
- Step 6: Confirm (show summary, Enter triggers Submit, Esc cancels)

**Event handler:**
`pub fn handle_gate_wizard_event(state: &mut GateWizardState, key: KeyEvent) -> GateWizardAction`

- Esc at any step → `GateWizardAction::Cancel`
- If `auto_skip_msg` is Some, any keypress clears it and advances to next step → `Continue`
- Step 0-1: Char → append to current buffer, Backspace → remove last char, Enter → validate and advance
- Step 2: Up/Down navigate extends_list_state, Enter selects and advances
- Step 3: Up/Down navigate includes_list_state, Space toggles entry in selected_includes, Enter confirms and advances
- Step 4: Criteria sub-step dispatch:
  - Name: text input; blank Enter on AddAnother moves to step 5; non-blank creates new CriterionInput entry
  - Description: text input; Enter advances to Cmd
  - Cmd: text input; Enter (blank = None cmd) advances to AddAnother
  - AddAnother: `y` adds new criterion (back to Name), `n`/Enter advances to step 5
  - Edit mode: walk existing criteria (show pre-filled), `d` to delete from vec, Enter to keep, after all existing → AddAnother
- Step 5: Precondition dispatch:
  - Ask: `y` sets preconditions_active=true → Requires; `n`/Enter → advance to step 6
  - Requires: text input loop; blank Enter ends → Commands
  - Commands: text input loop; blank Enter ends → advance to step 6
- Step 6: Enter → `GateWizardAction::Submit(assemble_gate_input(state))`

**Assembly helper:**
`pub(crate) fn assemble_gate_input(state: &GateWizardState) -> GateWizardInput`
- slug: `state.fields[0].last().cloned().unwrap_or_default()`
- description: non-empty fields[1] → Some, else None
- extends: extends_list_state.selected() — index 0 = None, index N = available_gates[N-1]
- include: filter available_libs by selected_includes indices, extract name
- criteria: clone state.criteria
- preconditions: if preconditions_active and (requires or commands non-empty) → Some(SpecPreconditions { requires, commands }), else None
- overwrite: state.edit_slug.is_some()

**Draw function:**
`pub fn draw_gate_wizard(frame: &mut Frame, area: Rect, state: &mut GateWizardState)`
- Note: takes `&mut GateWizardState` (not `&`) because `render_stateful_widget` needs `&mut ListState`
- Layout: header (3 lines) | main content (fill) | hint/error bar (3 lines) — same as draw_wizard
- Header: "Gate Wizard" (or "Edit Gate: {slug}" in edit mode) with step indicator "Step {N+1}/7"
- Main content varies by step:
  - Steps 0, 1: Paragraph showing current text input with cursor
  - Step 2: List widget with highlight for extends selection
  - Step 3: List widget with checkmarks for includes selection
  - Step 4: Current criteria sub-step text input + list of already-entered criteria above
  - Step 5: Precondition prompt or text input depending on sub-step
  - Step 6: Summary paragraph showing all collected values (name, description truncated, extends, includes comma-joined, criteria count, preconditions summary)
- Hint bar: step-specific key hints ("Enter: advance | Esc: cancel"), or error in red (Color::Red) if state.error is Some
- Auto-skip message: if auto_skip_msg is Some, show it in main content area with "(press any key)" hint
- Follow wizard.rs styling exactly: `Style::default().bold().reversed()` for highlight, `.dim()` for hints

**parse_slash_cmd:** Modify to handle parameterized `gate-edit` command BEFORE the COMMANDS table lookup:
```rust
pub fn parse_slash_cmd(input: &str) -> Option<SlashCmd> {
    let trimmed = input.trim().strip_prefix('/').unwrap_or(input.trim()).trim();

    // Parameterized commands first (cannot use COMMANDS table due to carried data).
    if let Some(rest) = trimmed.strip_prefix("gate-edit") {
        let arg = rest.trim().to_string();
        return Some(SlashCmd::GateEdit(arg));
    }

    // Exact-match lookup.
    let lower = trimmed.to_lowercase();
    COMMANDS.iter().find(|(name, _)| *name == lower).map(|(_, cmd)| cmd.clone())
}
```

Note: `strip_prefix("gate-edit")` already ensures case-sensitive matching. The lowercase fallthrough handles the COMMANDS table. The `GateEdit("")` case (no slug argument) will be handled by `execute_slash_cmd` returning an error string "Usage: /gate-edit <slug>".

**tab_complete:** The existing function works by prefix-matching against COMMANDS. Since "gate-wizard" is in COMMANDS, `/gate-w` will autocomplete to `/gate-wizard`. For `/gate-e`, it should match "gate-edit" — add a special case:
```rust
pub fn tab_complete(input: &str) -> Option<String> {
    let lower = input.trim().to_lowercase();
    if lower.is_empty() { return None; }

    // Parameterized command prefixes (not in COMMANDS table).
    if "gate-edit".starts_with(&lower) {
        return Some("/gate-edit ".to_string()); // trailing space for argument
    }

    let matches: Vec<&str> = COMMANDS.iter()
        .filter(|(name, _)| name.starts_with(&lower))
        .map(|(name, _)| *name)
        .collect();
    matches.first().map(|name| format!("/{name}"))
}
```
