**1. check_preconditions()**

```rust
pub fn check_preconditions(
    preconditions: &SpecPreconditions,
    last_gate_passed: impl Fn(&str) -> Option<bool>,
    working_dir: &Path,
    cli_timeout: Option<u64>,
    config_timeout: Option<u64>,
) -> PreconditionStatus
```

Implementation:
- Evaluate ALL `requires` entries (no short-circuit): for each slug, call `last_gate_passed(slug)`. Map `None` to `false` (conservative: no history = not passed). Collect into `Vec<RequireStatus>`.
- Evaluate ALL `commands` entries (no short-circuit): for each command, call `evaluate_command(cmd, working_dir, timeout)` where `timeout = resolve_timeout(cli_timeout, None, config_timeout)`. On Ok(result), use `result.passed`. On Err, mark as `passed: false` with output "spawn failed: {e}". Capture combined stdout+stderr as `output` via head+tail truncation from the GateResult (use existing `stdout` + `stderr` concatenation). Collect into `Vec<CommandStatus>`.
- Return `PreconditionStatus { requires, commands }`.

Evaluation order: requires first (cheap history lookups), then commands (expensive shell execution), per locked decision.

Add necessary imports at the top of the file: `SpecPreconditions`, `PreconditionStatus`, `RequireStatus`, `CommandStatus` from `assay_types`.

**2. evaluate_all_resolved()**

```rust
pub fn evaluate_all_resolved(
    spec_name: &str,

    gate_section: Option<&GateSection>,
    working_dir: &Path,
    cli_timeout: Option<u64>,
    config_timeout: Option<u64>,
) -> GateRunSummary
```

Implementation approach — thread source through evaluate_criteria:
- Modify the PRIVATE `evaluate_criteria` function signature to accept `Vec<(Criterion, Enforcement, Option<CriterionSource>)>` instead of `Vec<(Criterion, Enforcement)>`. This adds the source as the third tuple element.
- In `evaluate_criteria`, when building each `CriterionResult`, set `source` to the provided value from the tuple.
- Update ALL 4 existing callers of `evaluate_criteria` (`evaluate_all`, `evaluate_all_with_events`, `evaluate_all_gates`, `evaluate_all_gates_with_events`) to pass `None` as the third tuple element for each criterion. This preserves backward compat — existing paths produce `source: None`.
- In `evaluate_all_resolved`, map each `ResolvedCriterion` to `(rc.criterion.clone(), resolve_enforcement(rc.criterion.enforcement, gate_section), Some(rc.source.clone()))` and call `evaluate_criteria`.
- Pass `&[]` for events (resolved evaluation does not use event-based criteria).

Add necessary imports: `ResolvedCriterion`, `CriterionSource` from `assay_types`.
