<objective>
Change `detect_default_branch()` from infallible (`-> String`) to fallible (`-> Result<String>`) so that repos without a configured remote HEAD produce an actionable error instead of silently falling back to `"main"`.

Purpose: The silent fallback to `"main"` is a bug — it creates worktrees based on the wrong branch in repos where the default branch is named differently (e.g., `master`, `develop`). Actionable errors guide users to fix their git config.

Output: Modified `detect_default_branch()` and its single callsite in `create()`, plus integration tests proving the error path and bypass path.
</objective>

  <name>Task 1: Make detect_default_branch() return Result</name>
  <files>crates/assay-core/src/worktree.rs</files>
  <action>
Change `detect_default_branch()` signature from `-> String` to `-> Result<String>`.

Replace the current implementation (which falls back to `"main".to_string()`) with:

```rust
fn detect_default_branch(project_root: &Path) -> Result<String> {
    git_command(&["symbolic-ref", "refs/remotes/origin/HEAD"], project_root)
        .ok()
        .and_then(|output| {
            output
                .strip_prefix("refs/remotes/origin/")
                .map(|s| s.to_string())
        })
        .ok_or_else(|| AssayError::WorktreeGitFailed {
            cmd: "git symbolic-ref refs/remotes/origin/HEAD".to_string(),
            stderr: "Could not detect default branch. Run `git remote set-head origin --auto` \
                     or set `init.defaultBranch` in git config, or pass base_branch explicitly."
                .to_string(),
            exit_code: None,
        })
}
```

Update the single callsite in `create()` from:
```rust
let base = base_branch
    .map(|s| s.to_string())
    .unwrap_or_else(|| detect_default_branch(project_root));
```
to:
```rust
let base = match base_branch {
    Some(b) => b.to_string(),
    None => detect_default_branch(project_root)?,
};
```

The `?` propagates the error since `create()` already returns `Result`.

  </verify>
