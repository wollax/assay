**SAFE-02: TOCTOU-safe auto-promote (pipeline.rs:1130-1195)**

The simplest correct fix: after `promote_spec` returns `Err`, re-read the current status. If it's already at the target (`Verified`), treat as success with an info log. If it's a genuine error (IO, corrupt TOML), keep the existing `warn!`.

Replace the `Err(e)` arm at pipeline.rs:1174 with:
```rust
Err(e) => {
    // TOCTOU guard: another thread/process may have already promoted.
    // Re-read the spec to check if it's already at the target status.
    match crate::spec::load_feature_spec(spec_toml_path) {
        Ok(current) if current.status == assay_types::feature_spec::SpecStatus::Verified => {
            info!(
                spec = %spec_name,
                "auto-promote: spec already at target status Verified (concurrent promotion)"
            );
            // Still record auto_promoted on the session
            // (same with_session block as the Ok arm)
        }
        _ => {
            warn!(
                spec = %spec_name,
                error = %e,
                "auto-promote failed (non-fatal); continuing pipeline"
            );
        }
    }
}
```

Extract the `with_session` call into a helper closure or duplicate it in both arms (the Ok arm at :1158 and the "already at target" arm) to record `auto_promoted = true`.

**SAFE-03: Stderr capture in crash errors (pipeline.rs:422-474, 950-974)**

1. Add a `stderr_buffer` field to `StreamingAgentHandle`:
   ```rust
   pub struct StreamingAgentHandle {
       pub relay: std::thread::JoinHandle<i32>,
       pub pid: u32,
       pub stderr_buffer: Arc<Mutex<String>>,
   }
   ```

2. In `launch_agent_streaming`, change `stderr(Stdio::inherit())` to `stderr(Stdio::piped())`.

3. After spawning the child and extracting `child.stderr.take()`, spawn a second thread that reads stderr line-by-line into the `Arc<Mutex<String>>` buffer, capped at 4096 bytes (keep tail, trim head — the end of stderr is most useful for crash diagnosis).

4. In the crash error path at pipeline.rs:~962-974, read the stderr buffer and include it:
   ```rust
   let captured_stderr = agent_handle.stderr_buffer.lock()
       .map(|s| s.clone())
       .unwrap_or_default();
   let stderr_info = if captured_stderr.is_empty() {
       String::new()
   } else {
       format!("\n\nCaptured stderr:\n{captured_stderr}")
   };
   // Include stderr_info in the PipelineError message and recovery fields
   ```

**PITFALL (from RESEARCH.md):** Stdout and stderr must be read on separate threads to avoid pipe deadlock. The stdout relay thread already runs on its own thread. Stderr gets its own reader thread. The stderr thread can be a simple `spawn` that reads to the shared buffer — no need to join it explicitly (it will terminate when the pipe closes at child exit).
